# Milestone management

Inspect existing GitHub state before proposing changes. Give each milestone one clear outcome, bounded scope, and verifiable completion condition. Treat local configuration as guidance, not authority over existing remote state.

Read [references/full-guide.md](references/full-guide.md) when creating milestones, assigning issues, or running a full progress audit.
