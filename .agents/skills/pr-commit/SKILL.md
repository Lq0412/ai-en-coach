# Commit and PR

Inspect repository history and templates before applying defaults. Keep each commit and PR single-purpose, reviewable, and reversible. Never include unrelated user changes.

For commit types, scopes, splitting rules, and the four-part PR template, read [references/full-guide.md](references/full-guide.md) only when producing the corresponding artifact.
