# Branch and commit cadence

Inspect the repository's current conventions first. Keep branches single-purpose, short-lived, and safe to review. Commit at coherent, reversible checkpoints; avoid noisy checkpoint commits unless explicitly requested.

Read [references/full-guide.md](references/full-guide.md) when exact branch lifecycle rules, WIP policy, or the complete pre-push checklist is needed.
