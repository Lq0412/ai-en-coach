# Issue standard

Make each issue independently deliverable and verifiable. Inspect repository templates and labels before inventing defaults. Include context, scope, acceptance criteria, and dependencies only when relevant.

Read [references/full-guide.md](references/full-guide.md) when creating the full issue body, choosing labels, or deciding how to split a large task.
