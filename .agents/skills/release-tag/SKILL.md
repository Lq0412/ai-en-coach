# Release and tag

Inspect existing tag and release conventions first. Confirm the intended version and target commit before publishing. Produce concise release notes covering user-visible changes, fixes, compatibility concerns, and rollback information when relevant.

Read [references/full-guide.md](references/full-guide.md) for exact versioning, tag commands, release-note templates, and rollback steps.
