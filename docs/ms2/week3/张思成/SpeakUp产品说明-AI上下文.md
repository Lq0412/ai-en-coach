# SpeakUp 产品说明：给 AI 的上下文文档

> 用途：当需要让 AI 快速理解 SpeakUp 要做什么产品时，优先把本文作为上下文输入。  
> 当前口径日期：2026-07-21  
> 产品原型：`prototype/`  
> 重要参考：MS1 产品决策、超级 Agent 入口论证、长期 Agent 方向讨论、当前可交互原型。

## 1. 一句话定义

SpeakUp 是一个以语音和文字对话为主入口的 AI 英语口语教练，当前核心能力是帮助中国用户模拟外企、出海公司、海外岗位和国际化团队中的英文沟通场景，尤其是外企英文面试；同时它也可以扩展到日常口语、雅思口语、职场会议、客户沟通和轻量聊天。

SpeakUp 的目标不是做一个泛泛的英语聊天机器人，而是让用户在真实任务前练得更像真实场景，在任务后得到可复盘、可重练、可积累的反馈。

## 2. 产品本质

SpeakUp 的本质是：

```text
用户用中文或英文说出真实英语目标
  -> AI 理解目标和上下文
  -> 调用合适的练习能力
  -> 陪用户完成语音/文字对话
  -> 基于用户原话生成反馈
  -> 支持同题复练、错题沉淀和长期记忆
```

它更像一个长期认识用户的职业英语联系人，而不是一个传统课程 App。用户可以像给微信联系人发语音一样，说“我下周有个外企 PM 面试，帮我练一下”“帮我准备一次客户会议”“我想练雅思 Part 2”“刚才这段英文怎么说更自然”。AI 根据用户目标选择是直接给表达建议、开启模拟练习、创建面试计划、生成复盘，还是把结果写入长期记忆。

## 3. 当前主场景：外企英文面试模拟

现阶段最高优先级是外企英文面试口语陪练。

目标用户通常能读英文资料，也有基础表达能力，但在真实英文面试中容易出现这些问题：

- 无法在短时间内把项目经历、职责、行动和结果组织成结构化回答；
- 面对面试官追问时卡词、重复、偏题，或者缺少事实和数据证据；
- 背过标准答案，但问题稍微变化就不会自然表达；
- 分不清问题来自英语表达、内容结构、面试思路还是缺少准备；
- 练完后只得到泛泛建议，不知道下一次应该具体补什么。

SpeakUp 的面试训练闭环是：

```text
岗位 / JD / 简历或候选人背景
  -> 自动生成或配置面试计划
  -> 配置 1-4 位不同职责的面试官
  -> 选择一位面试官或群面模式开始模拟
  -> AI 用英文提问和追问
  -> 用户用语音或文字回答
  -> 保存问题、回答、转写和音频证据
  -> 生成逐题反馈和报告
  -> 用户针对同一问题再次复练
  -> 对比前后回答，沉淀弱点和进步
```

面试场景的价值不是“AI 会问问题”，而是把岗位、候选人背景、面试官职责、真实问答证据和同题复练版本绑定在同一个训练闭环里。

## 4. 可扩展场景

虽然当前主场景是外企英文面试，但产品内核不应写死为“面试软件”。SpeakUp 是一个可配置英语练习 Agent 平台，未来可以扩展到：

- 外企口语模拟：面试、入职自我介绍、绩效沟通、会议发言、跨国协作；
- 日常口语练习：旅行、点餐、问路、社交寒暄、生活服务；
- 雅思口语：Part 1 日常问答、Part 2 个人陈述、Part 3 深度讨论；
- 职场英语：客户会议、项目延期解释、产品演示、商务邮件口头化表达；
- 轻量聊天：用户只想低压力开口、润色一句话、练几分钟表达时，SpeakUp 可以作为英语对话伙伴。

这些扩展场景共用同一套底层能力：

- Agent 角色和任务目标；
- 对话策略和追问方式；
- 场景知识库或题库；
- 评分/诊断维度；
- 用户长期记忆；
- 练习历史、错题和复练。

## 5. 与普通 AI 聊天的区别

SpeakUp 不应该被做成“套壳 ChatGPT 语音聊天”。差异点是：

| 普通 AI 聊天 | SpeakUp |
|---|---|
| 回答当前问题 | 跟进用户真实英语目标 |
| 可以随便聊 | 围绕面试、职场、考试和口语提升形成任务闭环 |
| 建议常常泛泛而谈 | 反馈必须引用用户原话或具体表达 |
| 很少保留练习版本 | 保存历史回答、复练版本和进步证据 |
| 用户自己判断下一步 | AI 主动建议复练、面试计划、错题复习和表达积累 |
| 页面主要是聊天 | 聊天入口可以调用正式训练、Review、历史和知识库能力 |

一句话说，通用 AI 负责“回答你”，SpeakUp 负责“让你下次说得更好”。

## 6. 产品形态：像 GPT，但不是只有聊天框

界面形态接近 GPT：默认是一个对话入口，支持文字和语音输入，用户可以自然描述目标。

但 SpeakUp 不是把所有能力都塞进聊天气泡里。更合理的结构是：

```text
Agent 对话页
  = 默认首页、目标理解、长期上下文、任务编排

实时模拟页
  = 英文面试、雅思口语、角色练习等沉浸式语音会话容器

Review / 报告页
  = 逐题反馈、逐句改进、错题、复练和版本对比工作台

历史 / 记忆页
  = 面试计划、练习记录、表达弱点、进步证据和用户偏好

知识库
  = 岗位、JD、题库、表达素材、雅思题型和用户资料的支撑能力
```

Agent 的职责不是替代所有页面，而是替代用户手动找功能、填表、配置练习和回忆入口的过程。复杂任务可以通过 Action Card 或确认卡片启动，例如“开始模拟面试”“创建雅思 Part 2 练习”“复练刚才回答最差的问题”。

## 7. 典型用户路径

### 7.1 外企面试

```text
用户：我下周有个外企后端岗位面试，帮我练一下。
AI：询问缺失信息，读取简历或候选人资料。
AI：生成面试计划和面试官卡片。
用户：确认开始。
系统：进入语音模拟面试页。
AI 面试官：英文提问，并根据回答追问。
用户：语音回答。
系统：结束后生成报告，指出结构、证据、表达问题。
用户：选择最差一题同题复练。
系统：保存新版本并对比前后差异。
```

### 7.2 雅思口语

```text
用户：我想练雅思 Part 2，主题是 Describe a challenging project。
AI：生成 1 分钟准备 + 2 分钟回答的练习卡片。
用户：开始录音。
AI：按雅思口语维度反馈流利度、连贯性、词汇、语法和发音可理解度。
系统：给出更自然版本和下一次练习目标。
```

### 7.3 职场英语

```text
用户：我要和客户解释项目延期，帮我组织一下英文。
AI：先让用户用中文或英文讲清背景。
AI：给出可直接使用的英文表达。
AI：询问是否需要扮演客户追问两分钟。
用户：开始排练。
系统：保存这次真实事件，后续可跟进结果并复盘。
```

### 7.4 轻量聊天和日常口语

```text
用户：今天就想随便聊十分钟，帮我纠正明显错误。
AI：进入低压力聊天模式。
AI：不中断用户表达，只在合适节点给出少量自然纠正。
系统：把稳定问题沉淀到“我的表达”或错题复习中。
```

## 8. 核心产品原则

### 8.1 教练优先，陪伴其次

SpeakUp 可以温和、有陪伴感，但核心价值是帮助用户变强。聊天只是交互形式，学习目标才是内核。

### 8.2 目标驱动，不是功能驱动

用户不需要先知道“去哪个页面”。用户只需要说出目标，例如“练 Amazon behavioral round”“复盘刚才那轮面试”“练雅思 Part 3”。AI 负责理解、补齐信息并调用合适能力。

### 8.3 反馈必须可解释

不要只给裸分数。反馈要绑定用户原话，说明哪里有效、哪里缺证据、哪里表达不自然，并给出下一次具体怎么补。

### 8.4 进步必须可感知

产品要保存历史版本，让用户看到“上次怎么说，这次怎么说，哪里变好了”。复练和版本对比是核心闭环。

### 8.5 自由对话为形，任务目标为魂

用户感觉是在自然聊天，但 AI 内部有任务目标、追问策略和评分标准。正式训练不应机械地像题库考试，轻量聊天也不应失去教练能力。

### 8.6 Agent 不替代业务页面

聊天页负责理解和编排，实时语音、报告、复练、历史和设置仍然可以是独立页面。Agent 通过卡片和工具调用把用户带到正确位置。

## 9. 当前原型可以如何理解

`prototype/` 是当前可交互产品原型。它已经体现了以下方向：

- 整体界面接近移动端 GPT/Agent 对话产品；
- 有 Agent 首页和语音/文字入口；
- 支持创建模拟面试；
- 支持单面和群面概念；
- 支持面试官、面试计划、语音练习、报告、逐句改进、错题、复练和历史；
- 支持场景口语练习和角色对话的原型路径；
- 目前大量逻辑是静态或 Mock，用于验证信息结构和演示主链路，不等于真实后端能力已完成。

理解原型时要注意：它展示的是产品未来形态和主链路，不应把每个可点击入口都理解为当前真实交付范围。

## 10. 当前不应做成什么

SpeakUp 不应做成以下产品：

- 泛英语课程平台：不做完整课程体系、闯关、单词语法教学；
- 单纯发音评分工具：不以音素级纠音作为首要价值；
- 纯聊天陪伴 App：不能只有闲聊和情绪价值；
- 题库型面试模拟器：不能只是“出题、回答、打分、下一题”；
- 角色广场或 AI 朋友社区：当前不应把重点放在大量角色供给和社交；
- 只会打开页面的语音助手：Agent 应理解目标、组织任务、调用工具并沉淀结果。

## 11. AI 参与产品设计或开发时的默认假设

以后 AI 在阅读本文后参与设计、写 PRD、写代码或生成原型时，应默认以下口径：

1. 产品名是 SpeakUp。
2. 当前主方向是 AI 英语口语教练。
3. 首要高价值场景是外企英文面试模拟，尤其面向中国技术、产品、运营等国际化岗位候选人。
4. 产品界面类似 GPT/Agent：默认通过对话承接用户目标。
5. 语音是核心输入方式，文字是并列辅助方式。
6. 面试、雅思、场景口语、职场沟通和聊天都是 Agent 可调用的练习能力，不是彼此割裂的 App。
7. 核心闭环是“真实目标 -> 对话练习 -> 证据反馈 -> 同题复练 -> 长期记忆”。
8. 反馈必须基于用户原话，不要只给抽象建议或总分。
9. 复杂任务应通过确认卡片进入专门页面，不能让普通聊天页承担所有实时语音和报告工作。
10. 原型中很多功能是 Mock 或未来形态，做真实交付时要区分 Milestone 范围。

## 12. 推荐的短描述

如果只需要给 AI 一段短上下文，可以使用：

> SpeakUp 是一个语音优先、界面类似 GPT 的 AI 英语口语教练。它面向准备外企、出海公司、海外岗位和国际化协作的中国用户，当前核心场景是外企英文面试模拟，也可扩展到雅思口语、职场沟通、日常口语和轻量聊天。用户可以直接用语音或文字说出目标，AI 读取用户背景和历史表现，创建练习计划或直接进入对话训练。训练后，系统基于用户原话生成可解释反馈，支持逐句改进、同题复练、错题沉淀、版本对比和长期记忆。SpeakUp 不是泛聊天机器人，也不是课程平台；它的核心价值是持续帮助用户在真实英语任务中说得更清楚、更自然、更有说服力。
