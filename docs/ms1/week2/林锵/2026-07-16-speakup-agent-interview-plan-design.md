# SpeakUp Agent 面试计划设计

## 目标

Agent 收集完 JD、简历和练习重点后，先在对话中展示一份简洁的计划概要，再由“查看面试计划”进入独立的计划页。计划页通过横向滑动卡片展示多轮单面，每张卡片代表一个面试轮次，并包含该轮唯一的面试官。

## 产品模型

- 一个面试计划包含多个面试轮次。
- 单面模式下，每个轮次只有一位面试官。
- 每个轮次可以产生多个练习场次，问答、进度和报告独立保存。
- 群面模式保持现有模型：同一场次内有多位面试官。

## Agent 创建结果

- 创建完成消息只展示岗位名称、轮次数、面试官数、预计总时长和四轮名称。
- 不重复展示简历文件名、重点经历等确认阶段信息。
- 唯一主操作为“查看面试计划”。

## 面试计划页

- 顶部：只保留返回 SpeakUp 和居中的“面试计划”标题，不增加分隔线或重复岗位概要。
- 中部：横向滚动吸附卡片，一屏完整展示一张卡片；支持触摸滑动、鼠标滚轮和圆点定位。
- 卡片：占满页面主要可用高度，集中展示岗位、轮次名称、右上角问题进度、面试官头像、职位、姓名和一个主按钮。
- 主按钮状态：未开始为“开始本轮面试”，进行中为“继续本轮面试”，已完成为“再练一场”。
- 圆点指示当前轮次，支持点击和滑动切换。
- 轮次按顺序排列，但原型阶段允许自由进入任意一轮。

## 视觉约束

- 继续使用白底、浅灰卡片、黑色主按钮和淡青色点缀。
- 不使用蓝紫渐变、玻璃发光、积分星星、标签堆叠或卡片套娃。
- 卡片为单层结构；主要信息左对齐，面试官身份区域可以居中。
- 卡片接近占满可用页面高度，底部操作固定在卡片内部。

## 进入面试

点击轮次主按钮后直接进入实时语音面试，不再经过独立的面试前准备或麦克风检查页面。完成后生成异步报告，并在计划页和练习记录中更新该轮状态。

## 验收

- Agent 创建单面计划后生成四个轮次和四位面试官。
- 创建结果中可看到四轮概要和“查看面试计划”。
- 计划页可横向滑动四张轮次卡片。
- 从任意卡片进入面试时，使用对应轮次的面试官。
- 移动端比例下无横向页面溢出，下一张卡片保持可见提示。
