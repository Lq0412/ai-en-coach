## 二、各任务的完成标准

### T01 产品 Proposal 终审

关联：[实训营主仓 #9](https://github.com/1024XEngineer/XE3-ESL/issues/9)

- [ ] 确认面试是主路径，自定义角色是次级路径。
- [ ] 确认岗位可以自由输入，当前样例不是唯一支持岗位。
- [ ] 确认多位面试官按轮次出现，不做多人实时面试。
- [ ] 确认 #10、#11、#12、#14 与 #9 没有范围冲突。
- [ ] 确认商业化入口不属于本周期研发范围。

关联：[实训营主仓 MS1](https://github.com/1024XEngineer/XE3-ESL/milestone/1)

1. 留在 MS1，发布前必须完成；
2. 已完成，只需补验收和状态；
3. 移入 MS2；
4. 放入 Backlog 或取消。

必须重点判断：

- RAG #45：建议移出 MS1；
- 超级 Agent #40/#44：建议移入 MS2 或更后阶段；

## 三、分工建议

- T01 必须由能代表产品决策的人牵头，不能只交给开发同学自行判断。
- T05–T09 尽量由非 PR 作者担任 Reviewer；作者负责解释和修订。
- T10、T11、T12 存在顺序依赖，负责人可以不同，但接口必须提前对齐。
- T03、T14 适合由组长或统一维护 Milestone 的人负责，避免多人同时改状态。
- T15 可以提前写草稿，但不能提前打 Tag、发 Release 或关闭 Milestone。
