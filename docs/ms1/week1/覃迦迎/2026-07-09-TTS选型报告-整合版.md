# AI 面试教练 TTS 选型结论

> 日期：2026-07-09  
> 场景：面向海外求职者的 AI 面试教练  
> 结论用途：产品初期 TTS 供应商选型

## 结论

当前阶段建议**优先选择国内 TTS 模型**，先用国内方案完成 MVP 和真实用户验证；国外方案保留为后续国际化和高端音色储备。

推荐顺序：

| 优先级 | 方案 | 定位 |
|---|---|---|
| 首选 | Xiaomi MiMo TTS | MVP 主方案，限时免费，效果已初步认可 |
| 备选 1 | MiniMax TTS | 国内高质量付费备选，适合正式上线兜底 |
| 备选 2 | 阿里云 CosyVoice / 百炼 | 中英混合和声音克隆能力强，可作为低成本备选 |
| 备选 3 | Azure Speech | 稳定国际方案，适合作为长期 fallback |
| 储备 | Inworld / ElevenLabs | 海外部署或精品音色阶段再评估 |

一句话判断：**先用 MiMo 做主方案验证产品体验；如果限免结束或稳定性不足，切 MiniMax 或 CosyVoice；国外方案先作为调研结论保留，不作为当前默认接入方向。**

## 为什么优先国内方案

### 1. 当前目标是跑通产品闭环，不是追求理论最强音质

AI 面试教练早期最重要的是让用户能顺畅完成模拟面试，并获得有效反馈。TTS 只要做到英文自然、不卡顿、能表达面试官语气，就足够支撑 MVP。

目前国内模型中，MiMo、MiniMax 等已经有专门面向中英文语音的能力，试听效果可接受，不需要一开始就引入海外链路复杂度。

### 2. 国外方案有中转延迟和稳定性风险

Inworld、ElevenLabs 等国际方案理论上英语效果更强，但在国内开发和调用时可能涉及代理、中转、账号、支付、网络稳定性等问题。

对于实时对话产品，延迟和稳定性比单次音质更关键。一旦 TTS 响应不稳定，用户感知会非常明显。

### 3. 国内方案接入和试错成本更低

MiMo 当前 TTS 限时免费，适合低成本快速验证；MiniMax 等国内方案也更适合国内团队调试、排障和后续成本控制。

这能让团队把精力集中在核心体验上：面试流程、追问质量、反馈质量、用户留存，而不是过早处理跨境调用问题。

## 重点方案判断

| 方案 | 适配度 | 选择理由 | 风险 |
|---|---|---|---|
| MiMo TTS | 高 | 限时免费；支持流式、音色设计、声音克隆；试听效果可接受 | 限免不是长期价格，需确认正式计费和 SLA |
| MiniMax TTS | 高 | 国内语音大模型，情绪和自然度表现较好，适合作为付费稳定备选 | 成本高于限免 MiMo |
| CosyVoice / 百炼 | 中高 | 中英混合、声音克隆、开源生态和阿里云接入都有优势 | 英语面试真实感需盲测 |
| Azure Speech | 中高 | 稳定、文档成熟、国内可用性较好 | 声音真实感不一定最突出 |
| Inworld | 中 | 面向实时语音对话，适合海外部署 | 国内可达性和中转延迟需验证 |
| ElevenLabs | 中 | 英语声音质量强，适合精品音色 | 成本高，国内调用不稳定 |

## 国内已调研方案池

后续可以按下面顺序继续试用，不需要一次性全部接入：

| 试用顺序 | 国内方案 | 简要判断 |
|---:|---|---|
| 1 | Xiaomi MiMo TTS | 当前最值得先试，限时免费，能力覆盖完整 |
| 2 | MiniMax TTS | 英文自然度和情绪表达值得重点测试，适合作为正式付费备选 |
| 3 | 阿里云 CosyVoice / 百炼 | 中英混合、克隆和开源路线有优势，适合做长期成本优化 |
| 4 | 腾讯云 TTS | 成本低、云服务成熟，适合作为低成本兜底 |
| 5 | 百度智能云 TTS | 免费额度和接入友好，适合快速 demo 或临时验证 |
| 6 | 火山引擎 MegaTTS | 声音克隆能力强，适合后续做固定面试官音色 |
| 7 | 网易有道智云 | 英语词典发音有优势，适合单词/短句发音场景 |
| 8 | 科大讯飞 | 稳定、语音技术成熟，但更偏中文和方言优势 |
| 9 | 魔音工坊 / 标贝科技 | 音色资源和定制能力强，但 API 定价和接入需商务确认 |

当前建议重点测试前 3 个：**MiMo、MiniMax、CosyVoice**。如果要再加低成本兜底，可以补测腾讯云 TTS。

## 建议执行

1. **本周优先接入 MiMo TTS**，用真实面试台词测试英文自然度、延迟和稳定性。
2. **同步准备 MiniMax / CosyVoice 作为备用接口**，避免 MiMo 限免结束或服务不稳定时被卡住。
3. **保留 Azure / Inworld / ElevenLabs 调研结论**，等产品验证后再决定是否用于海外部署或付费精品音色。
4. 接入时统一封装 TTS Provider，避免后续切换供应商时重写业务逻辑。

## 最终建议

**当前选 MiMo TTS 作为 MVP 主方案，MiniMax 和 CosyVoice 作为国内备用，Azure 作为稳定国际备用。**

国外方案已经调研过，但不建议现阶段优先接入。等产品验证后，如果确实需要更强的英语真实感或海外部署，再评估 Inworld 和 ElevenLabs。
