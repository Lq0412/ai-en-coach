## 一、方向收敛

- **方向 Pivot**：从"高压场景英语口语"转向"外企英文面试陪练"，判断依据——面试场景用户目标明确、结果导向强、MVP 边界可控：[#2](https://github.com/1024XEngineer/XE3-ESL/issues/2)
- **竞品分析**：完成 Speak / ELSA / Duolingo Max / 流利说 / 扇贝口语 / 咕噜口语等 12+ 产品横向对比，提炼 Top10 未满足需求（"AI 对话机械"出现 102 次）
- **用户调研**：抓取 2000+ 条 App Store 评论，提炼用户痛点链——`不敢开口 → 听不懂真实语速 → 临场卡词 → 不知道说得是否自然 → 不知道如何复练`
- **市场规模**：完成中国及全球 AI 英语口语市场数据摘录与付费参照表：[#1](https://github.com/1024XEngineer/XE3-ESL/issues/1)

## 二、产品定义

- **产品概念定义 v0.2**：三层对话状态机（开放对话→定向追问→收束推进）、Agent 可扩展架构（内核不感知场景）、5 条产品原则、MVP 边界
- **MVP Proposal 定稿**：聚焦外企英文面试陪练，4 个子场景（自我介绍/行为面试/项目深挖/技术问答），简历上传进 LLM 上下文，10 条验收标准
- **功能全景与原型设计**：完整功能清单（P0 7 项 / P1 6 项 / P2 6 项），8 页原型逐页布局 + MS 归属标注，5 个待确认项

## 三、技术选型

- **前端**：React 18 + Vite + Tailwind CSS 裁定：[#7](https://github.com/1024XEngineer/XE3-ESL/issues/7)
- **后端**：根据导师反馈从 FastAPI 调整为 Go 方向，替换方案调研中：[#8](https://github.com/1024XEngineer/XE3-ESL/issues/8)
- **通信**：WebSocket 直连，WebRTC 后续升级：[#7](https://github.com/1024XEngineer/XE3-ESL/issues/7)
- **语音链路**：Qwen-Omni S2S 为主链路（<150ms 延迟）+ Qwen 3.7 ASR + MiniMax TTS 备选
- **数据库**：PostgreSQL + pgvector 倾向
- **TTS 选型**：国内外方案对比完成，整合版已定稿：[#4](https://github.com/1024XEngineer/XE3-ESL/issues/4)
- **ASR 选型**：Qwen 3.7 ASR 为主模型：[#5](https://github.com/1024XEngineer/XE3-ESL/issues/5)
- **对话大模型**：S2S vs 级联架构方案对比完成：[#6](https://github.com/1024XEngineer/XE3-ESL/issues/6)
