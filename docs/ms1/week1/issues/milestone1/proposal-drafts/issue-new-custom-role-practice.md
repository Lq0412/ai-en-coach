# [提案] 自定义角色与自由对话功能范围

> 关联：[Issue #9](https://github.com/1024XEngineer/XE3-ESL/issues/9) · [最新在线原型](https://speakup-preview.oopwqsnyzxm.chatgpt.site/)

## 动机 / 用户故事

模拟面试解决明确的求职任务，但同一套语音能力也可以支持用户在特定场景中自由练习。最新版原型已经提供“五步创建角色 → 角色预览 → 角色聊天”路径。该能力需要单独限定范围，避免它被误解为要建设通用 Agent 平台或角色社区。

典型用户故事：

- 作为一名想练特定场景的用户，我希望用几轮简单问答描述对话对象、关系和场景，而不是从空白提示词开始。
- 作为一名语音用户，我希望可以说出需求，也可以切换文本输入完成角色创建。
- 作为一名开始对话前想确认结果的用户，我希望预览角色名称、描述和声音，并能重新生成。
- 作为一名已经创建角色的用户，我希望用实时或点击说话方式进行私有对话，并在历史中继续或回看。

一句话概括：

**自定义角色是 SpeakUp 的次级私有语音练习模式，不是公开角色平台，也不替代面试主链路。**

## 目标用户

- 已理解 SpeakUp 语音交互，希望练习某个具体对话场景的用户。
- 不会编写复杂提示词，希望通过引导式问答创建角色的用户。
- 需要一个稳定对话对象，而不需要课程、社区或角色市场的用户。

## 现有做法及其不足

| 方案 | 优点 | 主要不足 |
|---|---|---|
| 空白聊天框 | 实现简单 | 用户不知道怎样定义场景，结果不稳定 |
| 开放提示词/Agent 配置 | 能力强 | 学习成本高，范围容易失控 |
| 引导式私有角色 | 开始容易，边界可控 | 场景深度和长期记忆有限 |

两个月选择引导式私有角色，只复用语音对话基础能力，不建设开放 Agent 平台。

## 本期范围与明确不做

### 两个月最终范围

#### 五步创建

- 使用五个简短问题逐步收集角色身份、与用户的关系、对话场景、风格和补充要求。
- 每一步同时显示中英文提示，降低理解门槛。
- 默认支持语音回答，并允许切换文本输入。
- 支持返回修改上一步；提交后生成角色草稿。

#### 角色预览

- 展示角色名称、关系/用途、简短人设和声音。
- 用户可以切换声音、修改必要字段或重新生成。
- 用户确认后才能开始角色聊天。

#### 角色聊天

- 支持实时连续对话与气泡/点击说话两种方式。
- 角色在会话中保持已确认的人设、关系和场景边界。
- 用户可以结束会话，并在历史中看到角色练习记录。
- MS3 至少支持一个私有角色；是否支持多个角色由后续容量评估决定。

### Milestone 实现深度

| Milestone | 做到什么程度 |
|---|---|
| MS1 | 原型展示五步创建、预览和聊天；全部可以使用固定数据 |
| MS2 | 不作为核心面试闭环的交付阻塞项，只复用已完成的语音能力 |
| MS3 | 真实支持创建和保存至少 1 个私有角色，完成一次可恢复的语音对话 |
| MS4 | 完成修改、重新生成、异常恢复、内容边界和历史可用性测试 |

### 明确不做

- 不做公开角色市场、关注、点赞、评论、分享或用户创作者体系。
- 不做课程体系、场景商城或大量预置角色运营。
- 不做能调用外部工具、执行任务或访问第三方账号的通用 Agent。
- 不做无限长期记忆、关系养成、亲密度、虚拟礼物或付费角色。
- 不做数字人、复杂头像生成、视频和口型动画。
- 不让角色创建优先于模拟面试主链路；MS2 核心闭环未完成时不扩大本模块。

## 关键决策与依据

### 决策 1：独立 Proposal，明确次级优先级

自定义角色拥有独立创建、预览、对话和历史状态，不能只写成模拟面试中的一个按钮。拆分 Proposal 有利于单独接受或延后，同时保护面试主链路的优先级。

### 决策 2：引导式五步问答，不暴露复杂提示词

五步问答把角色定义转化为普通用户能回答的问题，并通过预览让用户在开始前纠错。具体问题文案可以迭代，但最终需要覆盖角色身份、关系、场景、风格和补充约束。

### 决策 3：私有角色，不做社区

两个月内验证的是“用户能否快速得到一个可练习的对话对象”，不是内容分发和创作者经济。角色默认私有，可以减少内容审核、版权、社交和商业化范围。

### 决策 4：共享语音交互，不建设第二套会话系统

角色聊天与模拟面试共享录音、识别、播放、实时/点击说话、会话恢复和历史能力。区别只在角色上下文和报告要求，避免重复建设基础设施。

## 基本概念与信息结构

| 概念 | 定义 |
|---|---|
| Role Draft | 五步问答生成、尚未确认的角色草稿 |
| Custom Role | 用户确认并私有保存的角色 |
| Role Persona | 角色身份、与用户关系、场景、风格和补充约束 |
| Voice Profile | 角色对话使用的声音配置 |
| Role Session | 用户与一个 Custom Role 的连续语音会话 |

```text
Create Role
  -> Five Guided Answers
  -> Role Draft
  -> Preview / Edit / Regenerate
  -> Confirm Role
  -> Real-time or Push-to-talk Session
  -> History
```

模型提示词、内容安全策略、记忆长度和存储方式由架构 Proposal 决定。

## 原型 / Demo

- 在线原型：[SpeakUp 最新在线原型](https://speakup-preview.oopwqsnyzxm.chatgpt.site/)
- 相关页面：创建、创建角色、角色预览、角色聊天、历史。

最新版原型已经表达：

- 创建页将模拟面试作为主入口，将“创建一个角色”作为次级入口。
- 角色通过五步双语问题创建，支持语音和文本输入。
- 预览页显示 Marlowe 等演示角色、声音选择、重新生成和开始聊天。
- 聊天页支持实时与气泡/点击说话两种方式。

Marlowe、人设和旅行场景都是演示数据，不应成为默认固定角色。

## 验收标准

### Proposal 定稿验收

- [ ] 团队确认自定义角色为次级路径，并安排在 MS3 交付最小真实能力。
- [ ] 团队确认使用五步引导式创建，而不是复杂提示词或 Agent 配置。
- [ ] 团队确认角色默认私有，不做市场、社区、付费和关系养成。
- [ ] 团队确认角色聊天复用模拟面试的语音和历史基础能力。
- [ ] 团队确认 Marlowe 等角色仅为演示数据。

### 两个月产品行为验收

1. 用户可以通过五步语音或文本回答创建一个角色草稿。
2. 预览页能展示角色名称、人设、关系/用途和声音，并允许修改或重新生成。
3. 用户确认后可以用实时或点击说话模式完成一次角色对话。
4. 会话中的角色行为与已确认人设一致，不会无故切换身份或场景。
5. 录音、识别、模型或网络失败时有明确状态和重试入口。
6. 用户可以从历史找到角色练习记录；其他用户无法访问该私有角色。
