# [提案] 报告、逐句改进与错题复练功能范围

> 关联：[Issue #9](https://github.com/1024XEngineer/XE3-ESL/issues/9) · [最新在线原型](https://speakup-preview.oopwqsnyzxm.chatgpt.site/)

## 动机 / 用户故事

一次练习只能暴露问题，不能证明用户已经改进。报告如果只有一个总分或泛化建议，用户很难知道哪项目标没有完成、哪句话需要重说、下一次应该练什么。最新版原型已经把复盘拆成“练习报告 → 逐句改进 → 加入错题 → 再次练习”，本 Proposal 用于明确各层反馈的职责和验收边界。

典型用户故事：

- 作为一名刚完成模拟面试的用户，我希望先看到本轮目标达成情况和简短总结，而不是一个无法解释的总分。
- 作为一名表达不够准确的用户，我希望看到自己的原句、问题原因和更自然的优化表达。
- 作为一名需要口语练习的用户，我希望听到或跟读优化句，但不被一个不透明的音素分数打击。
- 作为一名持续准备面试的用户，我希望把表达、语法、词汇或发音问题加入错题，并知道它来自哪场练习。
- 作为一名想验证改进的用户，我希望从报告或错题直接再次练习。

一句话概括：

**反馈需要从会话目标落到真实原句，再转化为一个可执行、可回溯的复练项。**

## 目标用户

- 完成至少一次模拟面试或专项练习，需要具体改进建议的用户。
- 希望改善回答目标、流利度、词汇、语法和具体表达的用户。
- 希望跨多次练习管理薄弱项并继续复练的用户。

## 现有做法及其不足

| 反馈方式 | 优点 | 主要不足 |
|---|---|---|
| 单一总分/雷达图 | 快速、容易展示 | 依据不清，用户不知道下一步做什么 |
| 长篇 AI 总结 | 信息多 | 难核对、难行动、容易泛化 |
| 逐句纠错 | 具体 | 可能割裂面试目标，信息量过大 |
| 分层报告 + 代表原句 + 错题复练 | 兼顾全局和行动 | 需要控制反馈数量和归类规则 |

选择第四种方案。报告先回答“任务目标是否完成”，逐句页再回答“这句话怎样说得更好”。

## 本期范围与明确不做

### 两个月最终范围

#### 练习报告

- 展示练习时长、已完成目标数/总目标数和一段可理解的总结。
- 按本轮预设目标展示达成状态和原因，例如是否说明取舍、是否给出数据或结果。
- 展示语言表现，至少覆盖流利度、词汇和语法；评价需要附简短依据。
- 推荐 1-3 条最值得优先改进的代表句，避免把整段转录全部变成错误。
- 提供“加入错题”“开始错题回顾”“再次练习”等下一步入口。

#### 逐句改进

- 展示用户原句及必要的中文释义。
- 说明为什么需要优化，区分信息不完整、表达不自然、语法、词汇或发音问题。
- 展示优化表达和修改说明；优化句不能改变用户原意或虚构经历。
- 提供单词/短语发音提示以及整句跟读或朗读入口。
- 用户可以将该句加入错题，或直接返回当前练习再次回答。

#### 错题回顾

- 错题按表达优化、语法错误、词汇使用、发音四类组织。
- 每项保存来源练习、创建时间、原句/目标句和当前状态。
- 展示预计回顾时长，计算规则必须基于真实待练项数量。
- 用户可以进入单项练习，完成后标记为待复习或已掌握。
- 历史页可以查看面试与角色练习记录；只有产生相应反馈的练习才进入错题。

#### 成长记录

- 展示本周练习次数、累计时长、待复练数量等可解释指标。
- 根据近期未掌握错题推荐下一次练习，不建设复杂能力画像。

### Milestone 实现深度

| Milestone | 做到什么程度 |
|---|---|
| MS1 | 用演示数据串联报告、至少 1 条逐句改进、加入错题和再次练习；数据可 Mock |
| MS2 | 基于真实转录生成目标达成、语言表现和至少 1 条代表句；保存基础报告 |
| MS3 | 交付四类错题、来源回溯、单项复练、掌握状态、历史和基础成长统计 |
| MS4 | 完成生成失败、部分结果、重新生成、重复错题、数据一致性和跨会话测试 |

### 明确不做

- 不使用一个裸总分代表用户的完整面试能力。
- 不承诺对所有专业答案进行权威正确性判定，也不预测录用结果。
- 不做音素级准确率评分、口音等级或母语者相似度排名。
- 不把所有转录句子都标红，不一次展示大量低价值问题。
- 不做社交排行榜、打卡竞赛、公开成绩单或好友比较。
- 不做复杂技能树、勋章体系、好句社区或公开收藏库。
- 不让优化句虚构数字、职责、技术方案或业务结果。

## 关键决策与依据

### 决策 1：任务层与句子层分开反馈

面试报告先检查用户是否完成本轮目标，例如是否说明背景、个人贡献、判断依据和结果；逐句改进再处理流利度、词汇、语法和具体表达。这样可以避免“英语更自然”掩盖“回答没有内容”，也避免报告变成长篇语法课。

### 决策 2：目标达成数可以展示，裸总分不展示

“2/4 项目标达成”可以逐项核对，因此保留；单一的 78 分如果没有稳定规则，就不进入正式范围。流利度、词汇、语法等维度必须配简短依据，不能只显示颜色或数字。

### 决策 3：每次只推荐少量代表句

每场练习优先选择 1-3 条影响理解、专业表达或重复出现的句子。选择依据应能说明，避免用户被大量小错误淹没。用户仍可以查看完整转录，但它不等于全部需要进入错题。

### 决策 4：发音提示服务跟读，不承诺音素评分

最新版原型的逐词发音与整句跟读可以帮助用户练习优化句，因此进入范围。两个月内不建设或承诺音素级评分；发音类错题可以来自明显识别问题、用户主动添加或稳定规则，但不能伪装成高精度口音测评。

### 决策 5：错题是复练任务，不是收藏夹

错题必须包含来源、问题类型、目标表达和状态，并能进入下一次练习。只保存一条“好句”而没有练习动作，不属于本模块的核心价值。

## 基本概念与信息结构

| 概念 | 定义 |
|---|---|
| Session Goal | 一次练习希望用户完成的内容目标 |
| Goal Result | 某项目标已完成、部分完成或未完成，以及依据 |
| Language Feedback | 流利度、词汇和语法等语言表现及依据 |
| Sentence Improvement | 一组原句、问题原因、优化句和说明 |
| Retry Item | 从逐句反馈产生、可以再次练习的错题 |
| Retry Category | 表达优化、语法错误、词汇使用或发音 |
| Mastery Status | 待练习、待复习或已掌握 |

```text
Completed Session
  -> Goal Results + Language Feedback
  -> Select 1..3 Sentence Improvements
  -> Add Retry Item
  -> Practice / Read Aloud
  -> Update Mastery Status
  -> Recommend Next Practice
```

评分模型、提示词、相似度算法、音频片段存储和掌握判定阈值由架构 Proposal 决定。

## 原型 / Demo

- 在线原型：[SpeakUp 最新在线原型](https://speakup-preview.oopwqsnyzxm.chatgpt.site/)
- 相关页面：练习报告、逐句改进、错题回顾、历史。

最新版原型对应关系：

- 报告中的时长、目标 `2/4`、总结、情境达成分析、流利度/词汇/语法进入正式范围。
- “说明取舍未完成”“给出数据已完成”等状态是演示内容，实际目标随岗位和阶段变化。
- 逐句页的原句、中文释义、优化原因、优化表达、发音提示和跟读进入正式范围。
- 错题回顾的四类分组、来源面试和预计时间进入正式范围。
- 原型还需补充错题状态、完成一次复练后的变化，以及报告生成失败/部分结果状态。
- 历史页中的周练习、时长、错题数量可以保留；星星奖励如果没有规则，不进入验收。

## 验收标准

### Proposal 定稿验收

- [ ] 团队确认报告分为任务目标、语言表现和少量代表句三个层次。
- [ ] 团队确认保留可核对的目标达成数，不使用无法解释的裸总分。
- [ ] 团队确认逐句改进包含原句、原因、优化句、说明和跟读。
- [ ] 团队确认错题按表达、语法、词汇、发音四类组织，并保留来源和状态。
- [ ] 团队确认不做音素级评分、录用预测和社交排行。

### 两个月产品行为验收

1. 完成练习后，报告展示本轮目标及每项目标的状态和依据。
2. 流利度、词汇和语法评价均有可理解说明，不只显示数字或颜色。
3. 每场报告推荐 1-3 条来自真实转录的代表句，且不会虚构用户经历。
4. 逐句页展示原句、优化原因、优化句和说明，并可以播放/跟读。
5. 用户可以将句子加入正确类别的错题，并从错题回到单项练习。
6. 错题保存来源、时间和掌握状态；预计时长与待练项数量一致。
7. 报告或逐句生成失败时保留已完成会话，允许重试而不重复创建记录。
8. 历史与成长统计可以从真实会话和错题数据复核。
