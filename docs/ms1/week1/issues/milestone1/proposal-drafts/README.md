## 已统一的产品结论

- SpeakUp 的首要价值是英文技术相关岗位面试训练；目标岗位由用户自由输入，原型中的“产品经理”只是当前演示数据，不是产品唯一岗位。
- 两个月的主链路是：输入岗位背景 → 生成并编辑面试方案 → 确认分角色面试官 → 完整面试或专项练习 → 语音对话 → 报告 → 逐句改进/错题复练。
- 自定义角色是次级练习模式，用于复用语音对话能力并支持自由场景练习；它不能挤占模拟面试主链路的交付优先级。
- 简历属于可选的个性化上下文。最终支持 PDF、DOC、DOCX，最大 10 MB；MS2 先交付 PDF，MS3 再补 DOC/DOCX。
- 四位面试官是按轮次依次出现的角色化面试官，不是多人同时在线，也不扩展为视频数字人。
- 报告以目标达成、语言表现和真实原句改进为主；不使用无法解释的裸总分，也不承诺音素级发音评分。
- 账户需要承载简历、历史、报告和错题数据。昵称、头像、手机号属于低优先级基础资料；会员、权益中心和订单仅为原型中的商业化外壳，不进入本轮两个月研发验收。
- 原型表达未来两个月期望的产品形态，但 Milestone 决定每一阶段的真实实现深度；页面可点击不等于当前阶段已完成。

## Milestone 总体边界

| Milestone | 主要结果 | 允许 Mock 的内容 |
|---|---|---|
| MS1：战略决策与串联原型 | Proposal 定稿；原型覆盖两个月产品结构；用一个可替换岗位跑通面试主链路 | 用户、简历、语音、追问、报告均可使用演示数据，但要明确标识 |
| MS2：核心面试闭环可用 | 一种用户身份、一个岗位背景、1 份 PDF、真实语音问答、受控追问、基础报告 | 多面试官配置、DOC/DOCX、自定义角色、成长统计可继续 Mock |
| MS3：核心功能闭合 | 四轮面试与专项训练、角色化面试官、最多 3 份简历、错题复练、历史；交付最小自定义角色练习 | 商业化入口仍不实现 |
| MS4：质量与展示 | 异常恢复、隐私删除、稳定性、测试、部署和展示材料 | 核心链路不得依赖固定脚本 |
