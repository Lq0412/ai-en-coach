# MS1 表字段详细说明

> 日期：2026-07-14  
> 用途：详细说明 MS1 第一版数据库每张表的字段含义  
> 对应 SQL：`ms1-postgres-schema.sql`

## 1. 说明

本文是字段字典，用来补充说明每张表的字段用途、类型、是否必填、默认值和外键关系。

字段类型以 PostgreSQL 版本为准。

通用约定：

| 类型 | 说明 |
|---|---|
| `uuid` | 主键或外键 ID，默认使用 `gen_random_uuid()` 生成。 |
| `text` | 文本。 |
| `jsonb` | 结构化 JSON 数据。 |
| `timestamptz` | 带时区时间。 |
| `boolean` | 布尔值。 |
| `integer` | 整数。 |
| `bigint` | 大整数，文件大小使用。 |
| `date` | 日期，不含具体时间。 |
| `enum` | PostgreSQL 枚举类型。 |

通用时间字段：

| 字段 | 说明 |
|---|---|
| `created_at` | 记录创建时间。 |
| `updated_at` | 记录最后更新时间。第一版 SQL 只设置默认值，后续应用层或触发器负责更新。 |

## 2. `users`

用户账户表。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `id` | `uuid` | 是 | `gen_random_uuid()` | 用户唯一 ID。 |
| `email` | `text` | 是 | 无 | 登录邮箱，全局唯一。 |
| `password_hash` | `text` | 是 | 无 | 密码哈希，不能保存明文密码。 |
| `default_resume_id` | `uuid` | 否 | 无 | 默认简历 ID，外键指向 `resumes.id`。 |
| `deleted_at` | `timestamptz` | 否 | 无 | 注销或软删除时间。为空表示账户仍可用。 |
| `created_at` | `timestamptz` | 是 | `now()` | 创建时间。 |
| `updated_at` | `timestamptz` | 是 | `now()` | 更新时间。 |

约束：

| 约束 | 说明 |
|---|---|
| `email UNIQUE` | 同一个邮箱只能注册一个账户。 |
| `default_resume_id -> resumes.id ON DELETE SET NULL` | 默认简历删除后，用户默认简历置空。 |

## 3. `auth_sessions`

登录会话表。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `id` | `uuid` | 是 | `gen_random_uuid()` | 登录会话 ID。 |
| `user_id` | `uuid` | 是 | 无 | 所属用户，外键指向 `users.id`。 |
| `token_hash` | `text` | 是 | 无 | 登录 token 的哈希值，不能保存原始 token。 |
| `expires_at` | `timestamptz` | 是 | 无 | 登录态过期时间。 |
| `revoked_at` | `timestamptz` | 否 | 无 | 退出登录时间。为空表示尚未退出。 |
| `created_at` | `timestamptz` | 是 | `now()` | 创建时间。 |
| `updated_at` | `timestamptz` | 是 | `now()` | 更新时间。 |

约束：

| 约束 | 说明 |
|---|---|
| `user_id -> users.id ON DELETE CASCADE` | 用户删除后，登录态一起删除。 |
| `token_hash UNIQUE` | 一个 token 哈希只能对应一个登录会话。 |

## 4. `resumes`

简历文件索引表。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `id` | `uuid` | 是 | `gen_random_uuid()` | 简历 ID。 |
| `user_id` | `uuid` | 是 | 无 | 所属用户，外键指向 `users.id`。 |
| `file_name` | `text` | 是 | 无 | 用户上传时的原始文件名。 |
| `mime_type` | `text` | 是 | `application/pdf` | 文件类型。MS1 只允许 PDF。 |
| `file_size_bytes` | `bigint` | 是 | 无 | 文件大小，单位 byte。 |
| `storage_key` | `text` | 是 | 无 | PDF 在对象存储中的路径，不是公开 URL。 |
| `parse_status` | `resume_parse_status` | 是 | `uploaded` | 简历解析状态。 |
| `parse_error` | `text` | 否 | 无 | 解析失败原因。 |
| `is_default` | `boolean` | 是 | `false` | 是否为该用户默认简历。 |
| `parsed_at` | `timestamptz` | 否 | 无 | 解析完成时间。 |
| `deleted_at` | `timestamptz` | 否 | 无 | 删除时间。为空表示未删除。 |
| `created_at` | `timestamptz` | 是 | `now()` | 创建时间。 |
| `updated_at` | `timestamptz` | 是 | `now()` | 更新时间。 |

枚举：

| `parse_status` | 页面含义 |
|---|---|
| `uploaded` | 已上传，等待解析。 |
| `parsing` | 解析中。 |
| `available` | 解析成功，可使用。 |
| `failed` | 解析失败。 |

约束：

| 约束 | 说明 |
|---|---|
| `user_id -> users.id ON DELETE CASCADE` | 用户删除后，简历记录一起删除。 |
| `storage_key UNIQUE` | 一个对象存储路径只对应一份简历文件。 |
| `mime_type = 'application/pdf'` | MS1 只允许 PDF。 |
| `file_size_bytes <= 10485760` | 单份简历最大 10 MB。 |
| `resumes_one_default_per_user_idx` | 同一个用户最多一份未删除默认简历。 |

业务补充：

```text
每个用户最多保存 3 份未删除简历。
这个限制建议由业务层在上传前查询并判断。
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |

| `source` | `experience_source` | 是 | `parsed` | 来源是模型解析还是用户手动填写。 |
| `type` | `experience_type` | 是 | `project` | 经历类型。 |
| `title` | `text` | 是 | 无 | 经历标题，例如项目名称或岗位名称。 |
| `organization` | `text` | 否 | 无 | 公司、学校、组织或项目所属方。 |
| `start_date` | `date` | 否 | 无 | 开始日期。 |
| `end_date` | `date` | 否 | 无 | 结束日期，用于默认选最近经历。 |
| `sort_order` | `integer` | 是 | `0` | 在简历中的出现顺序。 |
| `background` | `text` | 是 | 空字符串 | 项目或经历背景。 |
| `responsibility` | `text` | 是 | 空字符串 | 用户个人职责。 |
| `achievements` | `text` | 是 | 空字符串 | 成果、指标或影响。 |
| `raw_extract_json` | `jsonb` | 否 | 无 | 模型解析出的原始结构化结果。 |
| `created_at` | `timestamptz` | 是 | `now()` | 创建时间。 |
| `updated_at` | `timestamptz` | 是 | `now()` | 更新时间。 |

枚举：

| `source` | 含义 |
|---|---|
| `parsed` | 从简历解析得到。 |
| `manual` | 用户手动填写。 |

| `type` | 含义 |
|---|---|
| `project` | 项目经历。 |
| `internship` | 实习经历。 |
| `work` | 工作经历。 |
| `education` | 教育经历。 |
| `other` | 其他经历。 |

约束：

| 约束 | 说明 |
|---|---|
| `user_id -> users.id ON DELETE CASCADE` | 用户删除后，经历一起删除。 |
| `resume_id -> resumes.id ON DELETE CASCADE` | 简历删除后，其解析经历一起删除。 |

| `extra_requirements` | `text` | 是 | 空字符串 | 用户补充要求。 |
| `experience_snapshot_json` | `jsonb` | 是 | 无 | 用户确认后的经历快照。 |
| `status` | `interview_plan_status` | 是 | `draft` | 面试计划状态。 |
| `created_at` | `timestamptz` | 是 | `now()` | 创建时间。 |
| `updated_at` | `timestamptz` | 是 | `now()` | 更新时间。 |

| `status` | 含义 |
|---|---|
| `draft` | 草稿，尚未确认面试官。 |
| `ready` | 已确认面试官，可以开始面试。 |
| `archived` | 已归档。 |

约束：

| 约束 | 说明 |
|---|---|
| `user_id -> users.id ON DELETE CASCADE` | 用户删除后，计划一起删除。 |
| `resume_id -> resumes.id ON DELETE SET NULL` | 简历删除后，计划仍保留。 |
| `source_experience_id -> experiences.id ON DELETE SET NULL` | 来源经历删除后，计划仍保留。 |

关键说明：

```text
experience_snapshot_json 是历史面试上下文的核心字段。
历史报告不能依赖当前 experiences 表，因为简历和解析结果可能被删除或修改。
```

| `report_status` | 含义 |
|---|---|
| `pending` | 尚未生成报告。 |

| 约束 | 说明 |

| `plan_id -> interview_plans.id ON DELETE CASCADE` | 计划删除后，场次一起删除。 |
| `interviewer_id -> interviewers.id ON DELETE CASCADE` | 面试官删除后，场次一起删除。 |
| `current_turn_index BETWEEN 0 AND 4` | 一场 MS1 面试最多 4 个有效 Turn。 |

| `transcript_status` | 含义 |
|---|---|

| `processing` | 转录中。 |
| `completed` | 转录完成。 |
| `failed` | 转录失败。 |

约束：

| 约束 | 说明 |
|---|---|
| `user_id -> users.id ON DELETE CASCADE` | 用户删除后，音频索引一起删除。 |

关键说明：

```text
audio_assets 只保存音频索引和元数据。
真实音频文件放对象存储，删除数据库记录前需要先删除对象存储文件。
```

## 10. `turns`

有效问答表。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `id` | `uuid` | 是 | `gen_random_uuid()` | Turn ID。 |
| `user_id` | `uuid` | 是 | 无 | 所属用户。 |
| `session_id` | `uuid` | 是 | 无 | 所属面试场次。 |
| `turn_index` | `integer` | 是 | 无 | 第几问，范围 1-4。 |
| `fixed_goal` | `fixed_turn_goal` | 是 | 无 | 本问固定目标。 |
| `question_text` | `text` | 是 | 无 | AI 生成的完整问题文本。 |
| `played_question_text` | `text` | 是 | 空字符串 | 用户实际听到的问题文本。被打断时可能短于完整问题。 |
| `was_interrupted` | `boolean` | 是 | `false` | AI 提问是否被用户打断。 |
| `answer_audio_asset_id` | `uuid` | 否 | 无 | 原回答音频 ID，外键指向 `audio_assets.id`。 |
| `answer_transcript` | `text` | 是 | 空字符串 | 用户原回答转录。 |
| `answered_at` | `timestamptz` | 否 | 无 | 有效回答完成时间。 |
| `created_at` | `timestamptz` | 是 | `now()` | 创建时间。 |
| `updated_at` | `timestamptz` | 是 | `now()` | 更新时间。 |

枚举：

| `fixed_goal` | 含义 |
|---|---|
| `project_overview` | 项目概览。 |
| `personal_contribution` | 个人贡献。 |
| `tradeoff_decision` | 方案取舍。 |
| `validation_result` | 验证与结果。 |

约束：

| 约束 | 说明 |
|---|---|
| `user_id -> users.id ON DELETE CASCADE` | 用户删除后，Turn 一起删除。 |
| `session_id -> interview_sessions.id ON DELETE CASCADE` | 场次删除后，Turn 一起删除。 |
| `answer_audio_asset_id -> audio_assets.id ON DELETE SET NULL` | 音频索引删除后，Turn 保留但音频引用置空。 |
| `turn_index BETWEEN 1 AND 4` | MS1 一场完整面试固定 4 问。 |
| `UNIQUE(session_id, turn_index)` | 同一场面试中第几问不能重复。 |

|---|---|
| `user_id -> users.id ON DELETE CASCADE` | 用户删除后，复练一起删除。 |
| `turn_id -> turns.id ON DELETE CASCADE` | Turn 删除后，复练一起删除。 |
| `feedback_item_id -> feedback_items.id ON DELETE CASCADE` | 反馈删除后，复练一起删除。 |
| `audio_asset_id -> audio_assets.id ON DELETE SET NULL` | 音频索引删除后，复练保留但音频引用置空。 |
| `attempt_index >= 1` | 复练次数从 1 开始。 |
| `UNIQUE(turn_id, attempt_index)` | 同一道题下第几次复练不能重复。 |

业务补充：

```text
只有 completed 的 interview_session 才允许创建 retry_attempts。
这个限制建议由业务层判断。
```

## 14. 字段设计关键点

```text
1. 所有用户私有数据表都带 user_id，便于权限隔离。
2. 简历 PDF 和音频文件不进数据库，只保存 storage_key。
3. interview_plans.experience_snapshot_json 保存历史上下文，避免简历删除影响历史报告。
4. current_turn_index 表示已完成有效 Turn 数量，不表示当前页面显示第几题。
5. turns 同时保存 question_text 和 played_question_text，用于支持 AI 被打断的场景。
6. feedback_items 和 turns 一一对应。
7. retry_attempts 只追加新版本，不覆盖原回答和旧复练。
```
