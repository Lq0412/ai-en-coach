# SpeakUp 超级 Agent 入口调研与立项论证

> 文档日期：2026-07-16<br>
> 产品：SpeakUp AI 英语面试陪练<br>
> 用途：产品评审、路演答辩、超级 Agent 入口立项论证<br>
> 核心问题：为什么需要增加一个支持语音与文字指令、能够调用 App 能力的超级 Agent 入口？

## 0. 执行摘要

SpeakUp 当前采用传统 App 导航模式：用户需要先理解产品的信息架构，再通过菜单和按钮进入面试复盘、继续练习、历史记录、专项训练等页面。

但用户真正想完成的通常不是“进入某个页面”，而是一个学习目标，例如：

> 帮我复盘上周那场产品经理面试，并练习其中回答最差的一道题。

超级 Agent 入口允许用户直接通过语音或文字表达目标，由 AI 识别意图、检索历史数据、选择工具、填充参数，并通过可确认的结果气泡将用户带到具体页面或下一项任务。

这一能力值得立项，主要基于五项判断：

1. 中国生成式 AI 用户已形成大规模，对话式交互的教育成本正在快速下降；
2. 学习者已经普遍形成向 AI 描述任务和获取辅助的习惯；
3. 软件功能普遍存在发现率不足的问题，功能增加会进一步放大导航复杂度；
4. 语音输入在移动端明显快于键盘输入，并且与英语陪练的核心场景天然一致；
5. Apple 和 Android 都在推动“把 App 能力开放给 AI 调用”的平台级架构。

因此，超级 Agent 入口不是单纯增加一个语音按钮，也不是追逐 Agent 概念，而是在降低用户从“产生学习目标”到“调用正确产品能力”之间的成本，并提升现有功能的可发现性、使用率和连续性。

## 1. 产品问题

### 1.1 用户必须先理解产品，才能使用产品

传统导航隐含要求用户完成以下转换：

```text
真实目标
  -> 判断这个目标对应哪个产品功能
  -> 回忆功能所在页面
  -> 逐层点击进入
  -> 再选择具体面试或题目
  -> 开始任务
```

例如，用户想“复盘上周的 Java 后端面试”，传统操作可能要求用户先进入历史模块、选择面试类型、找到具体记录，再打开 Review。

这类操作对产品团队而言很清晰，但用户并不天然理解“Review”“历史记录”“专项复练”等内部模块划分。用户理解的是目标，而不是功能树。

### 1.2 功能越多，传统入口的认知成本越高

SpeakUp 后续可能持续增加：

- 模拟面试；
- 面试 Review；
- 逐题回听；
- 同题复练；
- 高频问题归纳；
- 学习计划；
- 进步报告；
- 岗位专项练习；
- 简历和 JD 管理。

如果所有能力继续通过菜单扩展，产品会出现结构性矛盾：能力越丰富，用户越难判断应该使用哪个功能。

Pendo 基于匿名产品使用数据发布的功能采用报告显示，平均软件产品中约 **80% 的功能很少或从未被使用**。该数据不能直接代表 SpeakUp 的真实情况，但说明功能发现和采用不足是软件产品的普遍问题。[Pendo《2019 Feature Adoption Report》](https://go.pendo.io/rs/185-LQW-370/images/2019%20Feature%20Adoption%20Report%20Digital.pdf)

超级 Agent 入口的价值之一，就是让所有现有能力获得统一的自然语言发现入口。

## 2. 外部数据与行业证据

### 2.1 中国用户已经形成 AI 对话习惯

CNNIC 发布的第57次《中国互联网络发展状况统计报告》显示：

- 截至2025年12月，中国生成式人工智能用户规模达到 **6.02亿**；
- 较2024年底增长 **141.7%**；
- 普及率达到 **42.8%**，同比提高25.2个百分点。

这意味着用自然语言向 AI 描述问题和任务已经不再是少数技术用户的行为，而是在快速成为大众交互习惯。[CNNIC 第57次《中国互联网络发展状况统计报告》](https://www3.cnnic.cn/n4/2026/0304/c88-11549.html)

对于 SpeakUp 而言，用户不需要重新学习一种完全陌生的操作方式，而是可以把已经形成的 AI 对话习惯迁移到面试训练场景。

### 2.2 学习者已经普遍使用生成式 AI

英国高等教育政策研究所 HEPI 于2025年对1,041名全日制本科生开展的调查显示，**92% 的受访学生使用过生成式 AI**。[HEPI《Student Generative AI Survey 2025》](https://www.hepi.ac.uk/wp-content/uploads/2025/02/HEPI-Policy-Note-61.pdf)

该数据来自英国本科生，不能直接等同于 SpeakUp 的中国求职用户，但可以证明学习者正在形成“向 AI 提问、获取解释、反馈和学习辅助”的行为基础。

超级 Agent 入口顺应的是已经发生的用户习惯迁移：从寻找课程和功能，转向直接描述学习目标。

### 2.3 语音能够显著降低移动端输入成本

斯坦福大学、华盛顿大学和百度研究人员开展的移动端输入实验发现：

- 英语语音输入速度约为手机键盘输入的 **3倍**；
- 中文语音输入速度约为手机键盘输入的 **2.8倍**；
- 实验中的语音输入错误率也低于键盘输入。

这说明语音不是装饰性入口，而是能够降低移动场景输入成本的交互方式。[Stanford HCI：Speech Is 3x Faster than Typing](https://hci.stanford.edu/research/speech/)

语音对 SpeakUp 尤其合适，因为：

- 英语陪练本身就是以“说”为核心的产品；
- 用户已经理解并接受麦克风的使用场景；
- 练习结束后可以自然地继续说“复盘刚才的面试”；
- 用户不需要中断口语状态，转而输入和查找功能。

### 2.4 对话引导有机会减少学习系统中的操作摩擦

2026年发表于《Scientific Reports》的一项 AI 数字学习系统研究发现，实时对话式 Agent 与任务完成时间的下降存在显著关联。研究认为，对话式指导和澄清可以降低不确定性，减少不必要的页面转换，并加速任务执行。[Scientific Reports：AI-driven digital learning systems](https://www.nature.com/articles/s41598-026-41429-y)

该研究不能直接预测 SpeakUp 的实际提升幅度，但为“对话入口能够降低学习系统导航成本”提供了直接场景证据。

### 2.5 Apple 与 Android 正在推动 Agent 原生应用架构

Apple 的 App Intents 允许应用把自身动作和数据以结构化方式开放给 Siri、Spotlight、快捷指令和 Apple Intelligence。Apple 还提供 Interactive Snippet，使 AI 在识别和执行意图后展示结果、确认按钮或后续操作。这与 SpeakUp 计划采用的“识别指令—展示气泡—确认进入具体页面”模式高度一致。[Apple App Intents](https://developer.apple.com/documentation/appintents/making-actions-and-content-discoverable-and-widely-available)；[Apple Interactive Snippets](https://developer.apple.com/documentation/AppIntents/displaying-static-and-interactive-snippets)

Android 的 AppFunctions 则明确允许 App 将能力暴露为可被 Agent 和 AI 助手发现、选择和执行的工具。Android 官方将其描述为：用户可以通过自然语言完成复杂目标，减少逐步手动导航。[Android AppFunctions](https://developer.android.com/ai/appfunctions)

这说明“AI 理解意图并调用 App 能力”正在从单个产品实验，发展为移动操作系统支持的平台级交互范式。

## 3. 超级 Agent 入口的核心价值

### 3.1 从功能驱动转向目标驱动

传统产品路径是：

```text
用户寻找功能 -> 进入页面 -> 配置参数 -> 完成任务
```

超级 Agent 路径是：

```text
用户表达目标
  -> AI 识别意图与实体
  -> 调用历史数据和产品工具
  -> 展示结果或确认气泡
  -> 直接进入结果页面或继续执行任务
```

用户不再需要知道“哪个模块能做什么”，只需要知道“自己想完成什么”。

### 3.2 提升已有功能的可发现性和利用率

超级 Agent 不只是一个新增功能，还可以成为以下能力的统一分发入口：

- Review 某场历史面试；
- 继续上一次练习；
- 重练回答较差的题目；
- 查看近期进步；
- 归纳多场面试中的重复问题；
- 根据目标岗位开始新面试；
- 根据弱项生成当天练习计划。

因此，它有机会提升的不只是 Agent 自身使用率，还包括 Review、历史记录、复练和学习报告等既有模块的使用率。

### 3.3 将多个孤立功能组织成连续任务

普通语音搜索只能识别关键词并打开页面；超级 Agent 应理解目标并组合多个产品能力。

例如：

> 找出我最近三次面试中反复出现的问题，再让我练习最严重的那个。

Agent 需要执行：

```text
检索最近三场面试
  -> 读取三份 Review
  -> 归纳重复问题
  -> 确定最高优先级
  -> 生成结论与训练气泡
  -> 进入对应专项练习
```

这类跨模块编排能力，是超级 Agent 与普通语音搜索、固定快捷入口之间的核心差异。
