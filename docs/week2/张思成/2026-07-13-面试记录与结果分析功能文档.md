## 1. 文档目的

本文档定义 SpeakUp 的“面试记录与结果分析”模块，包括练习历史、面试情况概览、对话历史列表、单次会话详情、继续会话、面试报告总结、目标完成分析、语言表现及与错题本、复练页面的衔接。

## 2. 模块结论

本模块不是一个只展示历史列表的“记录页”，而是 SpeakUp 学习闭环的承接模块：

`完成或中断练习 → 保存记录 → 回看对话 → 查看诊断 → 进入错题复习或再次练习`

对用户而言，本模块需要回答五个问题：

1. 我练过什么？
2. 我练到哪里了？
3. 我当时具体说了什么？
4. 我做得怎么样、下次应该改什么？
5. 我现在可以继续、复习还是重新练？

## 3. 模块目标与范围

### 3.1 模块目标

- 让用户快速找到未完成或已完成的练习；
- 保留每次会话的关键上下文，支持中断恢复和事后回看；
- 将报告结论落到具体目标、具体语言问题和具体句子；
- 为用户提供明确的下一步行动：继续会话、再次练习、逐句改进、加入错题本；
- 让用户感知阶段性进步，而不是只看到一个缺乏解释的总分。

### 3.2 本期范围（P0）

- 练习历史总览和类型筛选；
- 模拟面试情况概览；
- 对话历史列表；
- 单次会话详情及对话记录；
- 进行中会话恢复；
- 已完成会话再次练习；
- 面试报告总结；
- 情境达成分析、目标完成情况、语言表现；
- 报告到逐句改进、错题回顾和复练的跳转；
- 加载、空数据、生成失败、无网络、记录缺失等异常状态。

### 3.3 暂不纳入本期

- 长期趋势图、能力雷达图和跨月成长报告；
- 音素级发音评分；
- 报告分享、导出 PDF；
- 用户手工编辑 AI 诊断结果；
- 多设备同时恢复同一会话的冲突合并；
- 排行榜和用户间能力对比。

## 4. 统一术语与状态

### 4.1 术语定义

| 术语 | 统一含义 | 示例 |
|---|---|---|
| 练习 | 产品内所有学习任务的总称 | 模拟面试、场景对话 |
| 模拟面试 | 练习类型之一，按岗位和面试流程组织 | 产品经理面试 |
| 场景对话 | 练习类型之一，按生活或职场场景组织 | 旅行开口聊 |
| 面试方案 | 用户创建的一组面试配置 | 岗位、简历、面试官、重点范围 |
| 轮次 | 一个面试方案下预先配置的练习单元 | 第 3 轮“研发协作交付” |
| 阶段 | 单轮中的专项练习类型 | 自我介绍、岗位八股、项目深挖 |
| 会话 | 用户一次实际进入并进行的练习记录；可中断、可恢复 | 今天 12:08 的 4 分钟会话 |
| 对话 | 一次会话内 AI 与用户的消息序列 | AI 提问、用户回答、AI 追问 |
| 报告 | 基于已产生的会话数据生成的结构化诊断 | 目标完成、语言表现、改进建议 |
| 错题 | 从报告或逐句分析中提取、可重复练习的问题项 | 表达、语法、词汇、发音问题 |

### 4.2 状态规范

| 对象 | 状态 | 含义 | 可用操作 |
|---|---|---|---|
| 练习/面试方案 | 未开始 | 已创建但没有有效会话 | 开始 |
| 练习/面试方案 | 进行中 | 至少有一次会话，仍有未完成轮次 | 继续、查看记录 |
| 练习/面试方案 | 已完成 | 规定轮次或结束条件已满足 | 查看报告、再次练习 |
| 会话 | 进行中 | 会话可从中断位置恢复 | 继续会话 |
| 会话 | 已完成 | 会话正常结束并保存 | 查看详情、再次练习 |
| 会话 | 异常结束 | 因网络、权限或服务异常未正常结束 | 恢复或重新开始 |
| 报告 | 生成中 | 会话已完成，报告异步生成 | 查看进度、稍后刷新 |
| 报告 | 已生成 | 报告内容可完整查看 | 查看、加入错题本、复练 |
| 报告 | 生成失败 | 报告任务执行失败 | 重新生成 |

统一要求：界面只使用“未开始、进行中、已完成”。当前原型中的“尚未开始”统一改为“未开始”，“已结束”统一改为“已完成”。“生成中/生成失败”仅用于报告状态，不能作为会话状态。

## 5. 用户入口与信息架构

### 5.1 使用入口

用户可以从以下位置进入本模块：

1. 底部主导航“历史” → 练习历史页；
2. 首页的“上次练习/继续练习”卡片 → 对应进行中会话；
3. 面试结束页 → 当前面试报告；
4. 面试情况概览 → 对话历史或面试报告；
5. 错题本中的“来源记录” → 对应报告或单次会话详情；
6. 系统通知“报告已生成” → 对应面试报告。

### 5.2 页面层级

```text
练习历史 D-01
├── 未开始练习 → 负责人 A/B/C 对应的开始页
└── 进行中/已完成模拟面试 → 面试情况概览 D-02
    ├── 对话历史 → 对话历史列表 D-03
    │   └── 单次会话详情 D-04
    │       ├── 继续会话 → 负责人 B 的面试执行页
    │       └── 再次练习 → 负责人 B 的面试轮次页
    ├── 面试报告总结 → 面试报告 D-05
    │   ├── 逐句改进 → 逐句改进 D-06
    │   ├── 加入错题本 → 负责人 E 的错题资产
    │   ├── 开始错题回顾 → 负责人 E 的错题练习页
    │   └── 再练一次 → 负责人 B 的面试轮次页
    └── 返回当前会话 → 负责人 B 的面试执行页
```

### 5.3 核心流程

#### 流程 A：继续未完成面试

`历史 → 选择“进行中”的模拟面试 → 面试情况概览 → 返回当前会话 → 从最近中断点继续`

#### 流程 B：回看某次会话

`历史 → 面试情况概览 → 对话历史 → 选择会话 → 单次会话详情 → 播放发音/查看翻译`

#### 流程 C：查看结果并复习

`历史 → 面试情况概览 → 面试报告 → 查看目标与语言表现 → 加入错题本 → 开始错题回顾`

#### 流程 D：完成后再次练习

`已完成会话详情或面试报告 → 再次练习/再练一次 → 选择原轮次或专项阶段 → 新建会话`

## 6. 页面功能说明

以下每个页面均按“模块目标、使用入口、页面结构、核心操作、状态变化、页面跳转、数据内容、异常情况、验收标准”九项说明。

---

### 6.1 D-01 练习历史

#### 1）模块目标

集中展示用户的模拟面试和场景对话，帮助用户继续未完成练习、查看已完成结果，并快速进入待复习错题。

#### 2）使用入口

- 底部主导航“历史”；
- 首页“查看全部练习”；
- 完成或退出练习后返回历史；
- 错题回顾结束后的“返回练习历史”。

#### 3）页面结构

1. 页面标题区：`MY PRACTICE / 练习历史`；
2. 数据概览区：练习数量、本周练习次数、累计对话时长、获得星标；
3. 今日待复习卡片：待复习总数及表达、语法、词汇、发音分类数；
4. 类型筛选区：全部、模拟面试、场景对话；
5. 练习记录列表：类型、单元/轮次数、名称、最近内容、状态、操作；
6. 底部主导航：创建、历史、我的。

#### 4）核心操作

- 点击“全部/模拟面试/场景对话”切换列表；
- 点击未开始记录的“开始”；
- 点击进行中记录的“继续”；
- 点击已完成记录进入面试情况概览或报告；
- 点击“开始复习”进入错题回顾；
- 下拉刷新最新记录和报告状态；
- 列表滚动到底后分页加载更多。

#### 5）状态变化

- 首次进入：骨架屏 → 加载成功或加载失败；
- 切换筛选：当前筛选高亮，列表按类型刷新；
- 未开始记录开始后：未开始 → 进行中；
- 完成最后一个规定轮次后：进行中 → 已完成；
- 错题完成后：待复习数量实时减少；
- 返回页面时保留用户上次的筛选和滚动位置。

#### 6）页面跳转

| 用户操作 | 目标页面 |
|---|---|
| 点击未开始的模拟面试 | 面试轮次页（B） |
| 点击进行中的模拟面试 | 面试情况概览 D-02；也可由“继续”直达当前会话（B） |
| 点击已完成的模拟面试 | 面试情况概览 D-02 |
| 点击场景对话记录 | 对应场景单元页或会话页（C） |
| 点击今日待复习 | 错题回顾页（E） |
| 点击“创建” | 创建入口（A/C） |
| 点击“我的” | 个人中心（E） |

#### 7）数据内容

- 用户维度：用户 ID、时区；
- 概览数据：练习总数、本周次数、累计时长、星标数；
- 记录数据：练习 ID、练习类型、标题、轮次/单元总数、最近内容、最近练习时间、状态、封面或角色；
- 复习数据：待复习总数、各错题类型数量；
- 筛选与分页：当前类型、游标、是否还有更多。

统计口径：

- “练习数量”按面试方案或场景方案去重，不按会话次数统计；
- “本周练习”按已产生有效用户语音/文本的会话统计；
- “对话时长”按有效会话时长求和，四舍五入到分钟；
- 0 秒或只打开未开始的会话不计入统计。

#### 8）异常情况

| 异常 | 页面处理 |
|---|---|
| 无任何记录 | 展示空状态、产品价值说明和“创建第一次练习”按钮 |
| 当前筛选无记录 | 展示“暂无该类型练习”，保留筛选器和“查看全部”入口 |
| 无网络 | 保留最近一次缓存，顶部提示“当前为离线数据”，提供重试 |
| 列表加载失败 | 显示局部错误，不影响顶部统计和底部导航 |
| 统计与列表暂时不一致 | 以列表状态为主，后台重新计算统计，不阻塞用户操作 |
| 记录已删除或无权限 | 移除失效记录并提示“该练习已不可用” |

#### 9）验收标准

- 用户从底部“历史”可在一次点击内进入本页；
- 三个筛选项互斥，选中态清晰，结果与类型一致；
- 未开始、进行中、已完成记录分别显示正确的操作文案；
- 点击进行中面试能恢复到正确面试和最近轮次；
- 统计字段口径与服务端定义一致；
- 空状态、无网络和加载失败均有明确解释和下一步操作；
- 页面返回后筛选和滚动位置不丢失。

### 6.2 D-02 面试情况概览

#### 1）模块目标

在进入具体记录前，汇总一个模拟面试方案的整体进度、累计数据、当前轮次，并提供对话历史、面试报告和返回当前会话三个入口。

#### 2）使用入口

- 练习历史中点击某条模拟面试记录；
- 面试报告点击“返回”；
- 对话历史点击“返回”；
- 当前面试退出时选择“查看面试情况”。

#### 3）页面结构

1. 顶部栏：返回、页面标题“面试情况概览”；
2. 状态与类型：进行中/已完成、模拟面试；
3. 面试信息：面试名称、最近练习轮次和阶段；
4. 进度区：整体进度、已完成轮次/总轮次；
5. 累计数据：累计对话时长、会话次数、获得星标；
6. 面试内容入口：对话历史、面试报告总结；
7. 当前轮次卡片：轮次名称、面试官、阶段、状态；
8. 主操作按钮：返回当前会话或再次练习。

#### 4）核心操作

- 点击对话历史查看全部会话；
- 点击面试报告查看结构化诊断；
- 点击“返回当前会话”恢复最近进行中会话；
- 已完成面试点击“再次练习”创建新会话；
- 点击返回回到练习历史。

#### 5）状态变化

- 进行中：展示当前轮次和“返回当前会话”；
- 已完成：整体进度为 100%，主按钮改为“再次练习”；
- 报告生成中：报告入口显示“生成中”，可进入等待页但不可重复提交；
- 报告生成失败：入口显示“生成失败”，允许重新生成；
- 新会话产生后，会话次数、累计时长和当前轮次更新。

#### 6）页面跳转

| 操作 | 目标页面 |
|---|---|
| 返回 | 练习历史 D-01 |
| 对话历史 | 对话历史 D-03 |
| 面试报告总结 | 面试报告 D-05 |
| 返回当前会话 | 面试执行页（B），恢复最近 checkpoint |
| 再次练习 | 面试轮次页（B），保留原面试配置 |

#### 7）数据内容

- 面试方案 ID、名称、类型、状态；
- 已完成轮数、总轮数、整体进度；
- 累计有效对话时长、会话次数、星标数；
- 最近轮次 ID/名称、阶段、面试官 ID/名称；
- 当前进行中会话 ID、恢复 checkpoint；
- 报告 ID、报告状态、更新时间。

整体进度计算：`已完成阶段权重之和 / 全部阶段权重之和`。若本期各轮权重相同，可先按已完成轮次占比计算，但页面必须使用同一口径，不能一处显示 38%，另一处显示 1/4（25%）且无解释。当前原型中“整体进度 38%”与“1/4 轮完成”存在口径差异，需要在研发前确定是按轮次还是按阶段计算。

#### 8）异常情况

- 面试记录不存在：返回历史并提示“该面试记录不存在或已删除”；
- 当前会话已在其他设备完成：刷新为已完成状态，不再允许继续旧 checkpoint；
- 报告未生成：展示生成状态和预计等待说明；
- 当前轮次配置已变更：使用会话创建时的配置快照回看，复练时使用最新配置并提示；
- 累计数据计算失败：内容入口仍可使用，统计区显示“暂未同步”。

#### 9）验收标准

- 页面展示的面试名称、状态、进度和当前轮次与记录一致；
- 进行中与已完成状态的主操作不同且正确；
- 报告页和对话历史页的返回均回到本页；
- 点击“返回当前会话”恢复正确会话和对话上下文；
- 不存在报告时不会展示空白报告；
- 进度百分比与轮次/阶段口径一致并可解释。

### 6.3 D-03 对话历史列表

#### 1）模块目标

按时间倒序展示同一个面试方案下的所有实际会话，让用户找到某一次记录并判断它是可继续还是可复练。

#### 2）使用入口

- 面试情况概览点击“对话历史”；
- 报告中的“查看本次对话”进入时，默认定位到对应会话。

#### 3）页面结构

1. 顶部栏：返回、标题“对话历史”、会话总数；
2. 页面说明：查看每次面试对话，进入详情后可继续或复练；
3. 会话列表：序号、时间、时长、会话标题、轮次、阶段、状态、进入箭头；
4. 分页加载区或加载更多状态。

#### 4）核心操作

- 点击会话卡片进入单次会话详情；
- 上拉加载更早会话；
- 从报告进入时自动定位并高亮对应会话；
- 点击返回回到面试情况概览。

#### 5）状态变化

- 默认按开始时间倒序；
- 进行中会话完成后自动更新为已完成；
- 会话异常中断时标为“异常结束”，详情提供恢复或重开；
- 新会话完成后插入列表顶部并更新总数。

#### 6）页面跳转

| 操作 | 目标页面 |
|---|---|
| 返回 | 面试情况概览 D-02 |
| 点击进行中会话 | 单次会话详情 D-04，展示“继续会话” |
| 点击已完成会话 | 单次会话详情 D-04，展示“再次练习” |
| 点击异常结束会话 | 单次会话详情 D-04，展示“尝试恢复/重新开始” |

#### 7）数据内容

- 会话 ID、面试方案 ID；
- 开始时间、结束时间、有效时长；
- 会话标题；
- 轮次序号/名称、阶段名称；
- 会话状态；
- 消息数量、是否存在可恢复 checkpoint；
- 报告或逐句诊断的关联 ID。

#### 8）异常情况

- 无会话：展示“还没有产生有效会话”，提供“开始练习”；
- 部分会话加载失败：已加载内容可用，并提供“加载更多失败，点击重试”；
- 时间跨时区：统一按用户当前时区展示，并在数据层保存 UTC；
- 时长缺失：显示“时长未记录”，不展示 0 分钟误导用户；
- 会话数据正在同步：卡片显示“同步中”，暂不允许继续，避免覆盖记录。

#### 9）验收标准

- 会话总数与列表数据一致；
- 列表默认按开始时间倒序且顺序稳定；
- 每张卡片至少显示时间、时长、轮次、阶段和状态；
- 不同状态进入详情后显示正确主操作；
- 返回后保持原滚动位置；
- 50 条以上数据可分页加载，不重复、不丢失。

### 6.4 D-04 单次会话详情

#### 1）模块目标

展示某次会话中的完整对话过程，让用户回看 AI 提问和自己的回答，并根据会话状态继续或再次练习。

#### 2）使用入口

- 对话历史列表点击任一会话；
- 报告或逐句改进点击“查看原对话”；
- 错题来源点击“查看上下文”。

#### 3）页面结构

1. 顶部栏：返回、会话标题、场景说明入口；
2. 会话元信息：轮次、阶段、日期、开始时间、有效时长；
3. 对话消息流：角色、时间点、原文；
4. 单句辅助操作：播放发音、显示/隐藏翻译；
5. 底部状态提示：可从中断位置继续/可再次练习；
6. 底部主操作：继续会话、再次练习或尝试恢复。

#### 4）核心操作

- 滚动查看完整对话；
- 点击“发音”播放当前句子的标准语音；
- 点击“翻译”显示或隐藏当前句译文；
- 点击“继续会话”从 checkpoint 恢复；
- 点击“再次练习”以原轮次、阶段和角色配置创建新会话；
- 点击场景说明查看本轮目标和面试官信息；
- 点击返回回到原对话历史位置。

#### 5）状态变化

- 进入页面：加载消息 → 定位到最后一条消息；
- 翻译默认收起，单句展开状态互不影响；
- 发音播放：未播放 → 缓冲中 → 播放中 → 已结束/失败；
- 进行中会话：显示“继续会话”；
- 已完成会话：显示“再次练习”；
- 恢复成功：旧会话继续写入；再次练习则创建新的会话 ID，不覆盖旧记录。

#### 6）页面跳转

| 操作 | 目标页面 |
|---|---|
| 返回 | 对话历史 D-03 |
| 继续会话 | 面试执行页（B），恢复原会话 |
| 再次练习 | 面试轮次页或直接进入对应阶段（B） |
| 查看场景说明 | 场景说明浮层或轮次详情（A/B） |
| 查看问题句分析 | 逐句改进 D-06 |

#### 7）数据内容

- 会话元数据：会话 ID、标题、轮次、阶段、角色、起止时间、时长、状态；
- 消息数据：消息 ID、发送方、原文、译文、时间戳、顺序号；
- 音频数据：音频地址、时长、有效期；
- 恢复数据：最近 checkpoint、上下文版本、最后确认消息 ID；
- 诊断关联：问题类型、报告项 ID、错题 ID。

隐私要求：会话记录属于用户私有学习数据；接口必须校验用户身份与记录归属，日志不得输出完整语音和简历敏感内容。

#### 8）异常情况

| 异常 | 处理 |
|---|---|
| 无对话消息 | 说明“本次会话未产生有效对话”，提供返回或重新开始 |
| 部分转写缺失 | 对应消息显示“转写失败”，保留音频播放入口 |
| 音频失效 | 禁用播放并提示“音频暂不可用”，不影响文本回看 |
| 翻译失败 | 当前句局部提示并提供重试 |
| checkpoint 失效 | 不覆盖旧记录，提示用户从该阶段重新开始 |
| 无麦克风权限 | 跳转执行页前说明授权方法，可先留在详情页 |
| 会话不属于当前用户 | 返回历史并提示无访问权限 |

#### 9）验收标准

- 消息严格按时间和顺序号排列，角色标识清晰；
- 每个“发音/翻译”只作用于当前句；
- 进行中会话显示“继续会话”，已完成显示“再次练习”；
- 继续会话不创建重复会话，再次练习必须创建新会话；
- checkpoint 恢复后不丢失已有消息；
- 无文本、无音频、无权限和翻译失败均可被正确处理。

### 6.5 D-05 面试报告总结

#### 1）模块目标

把练习结果转化为可理解、可执行的诊断，告诉用户做得好的地方、未完成的目标、具体语言问题以及下一步练习动作。

#### 2）使用入口

- 面试完成后自动进入或点击“查看报告”；
- 面试情况概览点击“面试报告总结”；
- 历史中的已完成记录进入概览后查看；
- 报告生成完成通知；
- 错题来源反向进入对应报告。

#### 3）页面结构

1. 顶部栏：返回、标题“面试报告总结”；
2. 报告头部：报告日期、面试用时、目标完成数/总数；
3. 总结：本次表现概述和下一次重点；
4. 情境达成分析：是否进入目标语境、回答策略是否合适；
5. 目标完成情况：已完成/未完成目标及证据与建议；
6. 语言表现：流利度、词汇、语法等；
7. 待改进项操作：加入错题本；
8. 底部行动：开始错题回顾、再练一次；
9. 可选入口：点击具体问题进入逐句改进。

#### 4）核心操作

- 阅读总结及各维度诊断；
- 点击目标或语言问题查看证据句和优化表达；
- 将单个或全部待改进项加入错题本；
- 开始错题回顾；
- 以原配置再练一次；
- 返回面试情况概览。

#### 5）状态变化

- 报告生成中：展示生成进度说明，禁止重复生成；
- 报告已生成：展示完整内容；
- 加入错题：未加入 → 提交中 → 已加入；重复点击不产生重复错题；
- 报告重新生成：保留旧版本，成功后切换最新版本；
- 复练完成后，可在报告顶部展示“已有更新练习”，但不覆盖历史报告。

#### 6）页面跳转

| 操作 | 目标页面 |
|---|---|
| 返回 | 面试情况概览 D-02 |
| 点击问题句/词汇项 | 逐句改进 D-06 |
| 开始错题回顾 | 错题回顾页（E） |
| 再练一次 | 面试轮次页（B），带入原面试配置 |
| 查看原对话 | 单次会话详情 D-04，并定位证据消息 |

#### 7）数据内容

- 报告 ID、会话 ID/面试方案 ID、版本、生成状态、生成时间；
- 有效面试时长；
- 目标总数、已完成目标数；
- 总结文本、下一次重点；
- 情境达成结论及证据；
- 目标项：目标名、状态、证据消息 ID、解释、建议；
- 语言项：维度、原表达、优化表达、解释、证据消息 ID；
- 错题关联状态；
- 生成模型和 rubric 版本（后台保存，用户端无需展示）。

报告原则：

- 不使用缺少解释的综合裸分；
- `★ 2/4`只表示完成了 4 个目标中的 2 个，不代表能力得分；
- 每条负面结论必须有证据或原句；
- 每条改进意见必须给出可执行建议；
- 报告必须基于本次会话，不能生成对话中不存在的事实；
- 用户语言内容和面试知识内容分开评价，避免“英语不流利”等同于“专业能力不足”。

#### 8）异常情况

| 异常 | 处理 |
|---|---|
| 报告生成超时 | 展示“生成时间较长”，允许离开，完成后通知 |
| 报告生成失败 | 展示失败原因的用户友好说明和“重新生成” |
| 会话内容不足 | 输出“数据不足”的有限报告，不编造完整结论 |
| 某维度无数据 | 显示“本次未覆盖”，不显示 0 分 |
| 加入错题失败 | 保留当前报告，局部提示并允许重试 |
| 报告版本过旧 | 仍可查看；复练时使用最新 rubric，并标明报告日期 |
| 内容安全拦截 | 对受影响部分做说明，其他可用维度正常展示 |

#### 9）验收标准

- 报告头部信息与对应会话一致；
- 目标完成数量等于目标项状态汇总；
- 每个未完成目标至少包含原因或改进建议；
- 每个语言问题可定位到原句或明确证据；
- 加入错题操作幂等，成功后状态立即可见；
- “返回、错题回顾、再练一次、查看原对话”跳转均正确；
- 生成中、生成失败、数据不足状态不会出现空白页；
- 报告不展示无法解释的综合裸分。

### 6.6 D-06 逐句改进

#### 1）模块目标

把报告中的抽象语言问题落到具体句子，提供原句、原因、优化表达、译文或说明，并支持用户跟读练习。

#### 2）使用入口

- 面试报告点击词汇、语法、表达或流利度问题；
- 单次会话详情点击被标记的句子；
- 错题详情点击“查看解析”。

#### 3）页面结构

1. 顶部栏：返回、可优化句子数量；
2. 用户原句及译文；
3. 为什么需要优化；
4. 优化表达及改进说明；
5. 逐词发音或重点词；
6. 主操作“跟读这个句子”；
7. 可选操作“加入错题本/查看原对话”。

#### 4）核心操作

- 查看原句和优化句对比；
- 播放整句或重点词发音；
- 跟读并完成一次录音；
- 加入错题本；
- 查看原对话上下文；
- 多个问题句时切换上一句/下一句。

#### 5）状态变化

- 跟读：未开始 → 录音中 → 识别中 → 已完成/失败；
- 加入错题：未加入 → 已加入；
- 多句任务完成后显示完成进度；
- 页面返回时保留报告的滚动位置。

#### 6）页面跳转

- 返回 → 原面试报告或错题详情；
- 查看原对话 → D-04，并定位到原消息；
- 加入错题本 → 数据写入 E 模块，不强制离开当前页；
- 完成跟读 → 留在当前页显示结果，用户可继续下一句。

#### 7）数据内容

- 问题项 ID、报告 ID、消息 ID；
- 原句、译文、问题类型；
- 原因说明、优化句、优化说明；
- 重点词和音频；
- 错题关联状态、跟读完成状态。

#### 8）异常情况

- 原消息已删除：仍显示报告快照，但“查看原对话”不可用；
- 发音音频失败：保留文本和跟读入口，允许重试；
- 麦克风无权限：说明授权方式，不影响阅读；
- 优化表达生成失败：展示原问题解释并允许重新生成；
- 无可优化句：返回报告并提示“本次没有需要逐句改进的内容”。

#### 9）验收标准

- 原句、优化句和问题解释对应同一消息；
- 从报告进入后定位到用户点击的问题项；
- 跟读只使用当前优化句；
- 加入错题操作不重复创建数据；
- 返回后报告位置和展开状态不丢失；
- 语音失败不影响用户阅读诊断。

## 8. 数据模型建议

### 8.1 核心对象关系

```text
User
└── PracticePlan（练习方案）
    ├── InterviewRound / SceneUnit（轮次/单元）
    ├── Session（会话，1:N）
    │   ├── Message（消息，1:N）
    │   └── Checkpoint（恢复点，0:1）
    └── Report（报告，1:N，按版本保存）
        ├── GoalResult（目标结果，1:N）
        ├── LanguageIssue（语言问题，1:N）
        └── MistakeItem（错题关联，0:N）
```

### 8.2 最小字段

| 对象 | 必需字段 |
|---|---|
| PracticePlan | id、user_id、type、title、status、total_rounds、completed_rounds、created_at、updated_at |
| Session | id、plan_id、round_id、stage、status、started_at、ended_at、duration_sec、checkpoint_id |
| Message | id、session_id、sequence、speaker、source_text、translation、audio_url、created_at |
| Report | id、plan_id、session_scope、status、version、summary、generated_at、rubric_version |
| GoalResult | id、report_id、goal_name、status、evidence_message_ids、analysis、suggestion |
| LanguageIssue | id、report_id、message_id、type、original_text、improved_text、reason |
| MistakeItem | id、user_id、source_type、source_id、issue_type、review_status、next_review_at |

### 8.3 数据保存规则

- 会话开始时立即创建 Session，避免异常退出后无记录；
- 每条已确认消息按顺序增量保存；
- checkpoint 至少在用户完成一轮回答后更新；
- 再次练习创建新 Session，不覆盖旧 Session；
- 报告按版本保存，重新生成不直接覆盖旧报告；
- 报告证据引用消息 ID，同时保存必要文本快照，避免消息内容变更后报告失真；
- 删除练习前需说明会同步删除或匿名化哪些会话、音频、报告和错题来源。

## 9. 异常与边界规则

| 场景 | 统一规则 |
|---|---|
| 无网络进入历史 | 展示缓存记录和离线标识；涉及写入的操作排队或禁止并说明 |
| 无记录 | 提供创建首次练习入口，不出现空白列表 |
| 会话进行中但无 checkpoint | 允许进入详情，但“继续”改为“重新开始本阶段” |
| 会话已完成但报告未完成 | 可回看对话，报告入口展示生成中 |
| 报告生成失败 | 不影响会话详情，允许用户主动重试 |
| 语音存在但转写缺失 | 显示音频与转写失败状态，不伪造文本 |
| 用户删除简历 | 历史会话和报告保留创建时的必要快照，不显示已删除简历原文 |
| 面试官/角色被删除 | 历史页显示角色快照；再次练习时要求重新选择可用角色 |
| 同一按钮重复点击 | 继续、再次练习、加入错题和重新生成报告均需防重复提交 |
| 用户跨时区 | 数据库存 UTC，页面按用户当前时区展示，并保留绝对日期避免歧义 |

## 10. 验收清单

### 10.1 主流程验收

- [ ] 用户能从底部导航进入练习历史；
- [ ] 用户能筛选全部、模拟面试、场景对话；
- [ ] 用户能从进行中记录进入正确面试概览；
- [ ] 用户能查看同一面试下的所有会话；
- [ ] 用户能打开单次会话并回看完整消息；
- [ ] 进行中会话能从正确 checkpoint 恢复；
- [ ] 已完成会话能创建一次新的复练会话；
- [ ] 用户能查看面试报告的总结、目标和语言表现；
- [ ] 报告问题能定位到具体句子；
- [ ] 待改进项能幂等加入错题本；
- [ ] 用户能从报告进入错题回顾和再练一次；
- [ ] 所有返回路径符合第 7.2 节规则。

- [ ] 非当前用户无法访问会话、音频和报告。

## 12. 协作边界

| 负责人 | 负责内容 | 与 D 的接口 |
|---|---|---|
| A：模拟面试创建与配置 | 面试方案如何生成、岗位/简历/面试官/轮次配置 | 向 D 提供面试方案、轮次、目标和角色快照 |
| B：模拟面试执行 | 面试如何进行、阶段切换、语音对话和结束 | 向 D 保存 Session、Message、Checkpoint，并接收继续/复练跳转参数 |

| E：学习资产与个人中心 | 错题、简历、账户、设置长期管理 | 接收 D 创建的错题，提供错题状态和来源回跳能力 |

## 13. 原型问题与待确认项

以下问题不阻塞本文档交付，但建议在进入研发前由团队统一确认：

1. 当前原型的“整体进度 38%”与“1/4 轮完成”口径不一致，需要确认是否按阶段加权；
2. 当前原型筛选“模拟面试”后仍展示场景对话卡片，仅增加“仅显示模拟面试”提示，研发版应真正过滤列表；
3. 当前原型使用“尚未开始、已结束”，需按本文统一成“未开始、已完成”；
4. 报告是按单次会话生成，还是按整个面试方案累计生成，需要明确；建议单次报告保留、方案概览做聚合；
5. `★ 2/4`应明确标注为“目标完成 2/4”，避免用户误认为能力评分；
6. 单次会话详情中的“发音”是播放标准 TTS 还是播放原录音，需要统一按钮文案；建议分别使用“标准发音”和“原录音”；
7. 对话原文与译文的默认语言需要根据实际练习语言显示，原型中的中文内容仅可作为占位数据；
8. “继续”是先进入面试情况概览还是直接进入面试轮次，需按入口区分：点击卡片进概览，点击卡片内“继续”直达当前会话；
9. 场景对话是否复用面试报告结构，需要 C、D 共同确定通用字段和场景专属 rubric。

## 14. 交付完成定义

当以下条件同时满足时，可认为 D 模块功能设计完成：

- 6 个页面的九项功能说明评审通过；
- 术语和状态被 A、B、C、E 共同采用；
- 全局跳转矩阵不存在无返回路径或死页面；
- 核心数据对象和负责人之间的输入输出已确认；
- P0 验收清单可以直接转成测试用例；
- 第 13 节待确认项已在原型或研发任务中给出结论。
