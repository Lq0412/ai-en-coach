## 1. 文档目的

本文从总 PRD、面试主链路细化稿和当前产品原型出发，梳理 SpeakUp 整个系统的功能范围、模块关系和产品边界，为后续领域建模、架构拆分、原型修改和迭代规划提供统一输入。

需要区分三类内容：

- **当前确认范围**：PRD 已明确、需要优先跑通并验收的功能。
- **原型终局能力**：原型已经展示，但不一定进入当前版本真实交付的功能。
- **后续扩展能力**：角色对话、实时通话、语言改进等可复用到其他口语练习场景的能力。

## 2. 整体产品结构

SpeakUp 可以理解为一个 AI 英语情境练习平台，模拟面试是当前最先落地的核心场景。

```text
SpeakUp
├── 用户与账户
├── 创建练习
│   ├── 模拟面试
│   └── 自定义角色练习
├── 语音对话
│   ├── 一问一答
│   ├── 对话气泡
│   └── 实时通话
├── 练习反馈
│   ├── 面试报告
│   ├── 逐题反馈
│   ├── 逐句改进
│   └── 错题复习
├── 历史记录
└── 个人资料与设置
```

当前产品优先跑通：

```text
创建面试计划
→ 配置面试官
→ 选择完整模拟或专项
→ 动态语音问答
→ 生成报告
→ 同题复练
→ 查看历史
```

## 3. 用户与账户模块

### 3.1 注册与登录

当前确认功能：

- 邮箱注册。
- 密码登录。
- 退出登录。
- 注销账号。
- 登录状态保护。
- 不同用户的数据隔离。

异常状态包括：

- 邮箱已经注册。
- 密码错误。
- 登录状态失效。
- 未登录访问受保护页面。

### 3.2 个人中心

原型已经展示：

- 用户头像和昵称。
- 累计练习时长。
- 星星或成长数据。
- 个人简历入口。
- 设置入口。
- 权益中心和订单入口。

当前版本保留头像、昵称、邮箱、个人简历和设置。权益中心、会员与订单暂不进入真实交付，原型中的手机号需要替换为登录邮箱。

### 3.3 设置与数据安全

- 修改头像和昵称。
- 查看隐私政策、服务条款和产品信息。
- 退出登录。
- 注销账号及二次确认。
- 注销后删除账号、简历、计划、场次、音频、转录、报告和复练记录。
- 音频和历史地址必须受到用户权限保护。

## 4. 候选人资料与简历模块

### 4.1 简历管理

- 每个账户最多保存 3 份简历。
- 当前只支持 PDF。
- 单份文件最大 10 MB。
- 支持设置默认简历、删除简历和切换简历。
- 展示上传和解析状态。

```text
上传中
→ 解析中
→ 可用 / 解析失败
```

### 4.2 整份简历解析

系统从整份简历中提取：

- 姓名。
- 学校、学历和专业。
- 工作经历。
- 实习经历。
- 项目经历。
- 技能。
- 个人优势。
- 成果和相关指标。

用户不需要先选择某一条具体经历。解析完成后先展示候选人资料摘要，用户可以直接继续，也可以展开修改字段。

解析失败时提供：

- 重新解析。
- 更换简历。
- 改为手动填写。

### 4.3 无简历路径

无简历时允许手动填写姓名、教育、工作/实习经历、项目经历、技能和个人优势。所有字段均可不填，缺失字段不能使用 mock 经历自动补齐。

### 4.4 示例体验

用户完全不填写岗位和候选人资料时，可以明确进入“示例体验”：

- 使用统一的示例岗位、JD 和候选人资料。
- 创建、面试、报告和历史均展示示例标识。
- 不让用户误以为示例事实来自自己的简历。
- 用户提供部分真实信息时，不得混入 mock 事实补齐。

## 5. 模拟面试模块

### 5.1 创建面试计划

创建流程分为三步：

```text
岗位信息
→ 候选人资料
→ 练习重点
```

岗位信息包括岗位名称和 JD：

- 只填岗位名称时，AI 自动生成可查看、可修改的 JD 摘要。
- 岗位名称和 JD 都填写时，以用户输入为准。
- 两者都不填写时进入示例体验。

练习重点支持预设标签多选和自由文本补充，两者都可以跳过。例如自我介绍、项目深挖、岗位能力、行为面试、英语表达和终面判断。

创建成功后：

- 生成一份面试计划。
- 保存岗位、最终 JD、候选人资料、练习重点和示例标记快照。
- 历史首页立即增加一张计划卡。
- 重复请求不能产生重复计划。

### 5.2 面试官配置

系统根据岗位自动配置 2—4 位面试官，用户不需要先选择预设面试官类型。

```text
岗位预设面试官
+ 最终 JD
+ 候选人资料
+ 练习重点
→ 一次轻量 AI 适配
```

每位面试官包含：

- 虚拟姓名。
- 职位和职责。
- 面试风格。
- 2—3 个关注点。
- 默认音色。
- 完整模拟环节。
- 2—5 个专属专项。

首次练习前允许编辑、添加、删除、排序面试官，编辑练习环节和专项，以及试听和更换音色。任意练习开始后，整组面试官和练习内容配置锁定。

音色可以继续更换，但只影响未来播放，不改变历史音频、报告或场次快照。

### 5.3 面试计划详情

- 使用面试官轮播，一次只展示一位面试官。
- 用户可左右切换，不要求按顺序完成。
- “完整模拟”作为主要操作。
- “更多练习”打开该面试官的专属专项面板。
- 每位面试官的完整模拟和专项都可以反复练习。
- 计划详情和面试官详情均提供历史入口。

### 5.4 完整模拟与专项练习

| 类型 | 默认时长 | 主问题上限 | 单题追问上限 |
|---|---:|---:|---:|
| 完整模拟 | 约 15 分钟 | 最多 6 个 | 最多 1 个 |
| 专项练习 | 约 8 分钟 | 最多 3 个 | 最多 1 个 |

用户当前不选择时长或问题数量。问题上限不是必须问满的配额，系统还需要结合剩余时间和目标覆盖情况决定是否继续。

### 5.5 开始前准备

选择练习后进入准备页，展示：

- 面试官姓名和职位。
- 当前练习类型。
- 预计时长。
- 完整模拟的环节或专项考察方向。
- 麦克风检查入口。

准备页不提前展示正式问题。

### 5.6 麦克风检查

- 说明需要麦克风的原因。
- 用户主动点击后请求权限。
- 展示输入设备和音量反馈。
- 完成一次不保存的短试音。
- 检查完成后由用户主动进入面试。
- 权限拒绝、没有设备、无有效声音或设备失效时提供修复入口。

## 6. 面试对话执行模块

### 6.1 一问一答主流程

```text
生成并保存问题
→ TTS 播放问题
→ 用户录音回答
→ 上传并保存音频
→ ASR 转录
→ 判断回答是否有效
→ 保存 Turn
→ 决定追问、换题或结束
```

每次只问一个主问题，不能在一句中堆叠多个问题。

### 6.2 问题生成依据

问题由以下信息共同决定：

- 最终 JD。
- 候选人资料。
- 面试官职责和关注点。
- 完整模拟当前环节或所选专项。
- 用户练习重点。
- 已经发生的对话。
- 剩余时间和问题数量。

问题优先来自“JD 要求与候选人经历的交集”，再体现当前面试官的职责和用户希望重点练习的内容。

系统开始时只生成隐藏的面试目标提纲和第一道具体问题，不一次生成整套固定题库。

### 6.3 对话目标控制

系统将每个目标标记为已覆盖、部分覆盖或未覆盖。每轮回答后，控制器根据目标覆盖情况、剩余时间、主问题数量和追问次数决定：

```text
FOLLOW_UP       追问当前问题
NEXT_OBJECTIVE  进入下一个目标
COMPLETE        结束面试
```

AI 只负责根据控制器选定的目标生成具体问题，不自行决定无限追问或延长面试。

### 6.4 常见问题类型

- 自我介绍。
- 求职动机和岗位匹配。
- 项目概览和个人贡献。
- 方案取舍和结果验证。
- 专业能力。
- 协作与冲突。
- 情景判断。
- 反思和成长。
- 候选人反问。

并非所有面试都必须从自我介绍开始，是否包含自我介绍由岗位、面试官职责和练习类型决定。

### 6.5 有效回答

回答只有满足以下条件才计入进度：

- 音频已经完整结束并保存。
- 转录可用。
- 不是空白、背景噪声或误触。
- 达到最低有效回答条件。

空白、过短或转录失败时不增加进度，保留当前问题并允许重新回答。

### 6.6 退出、恢复和提前结束

退出页面：

- 保留已经完成的 Turn。
- 丢弃当前未完成录音。
- 下次恢复到当前问题。
- 原 Session 可以继续。

提前结束：

- 封闭当前 Session。
- 保留已完成问答。
- 生成标记为“未完整完成”的阶段性报告。
- 原 Session 不能继续，但可以重新开始一场。

### 6.7 实时通话

原型已展示实时对话开关、连接状态、自动判断说话结束、自然打断和结束通话。当前 PRD 只确认保留“实时通话”入口，一问一答仍是当前主流程。

实时通话的进入时机、切换确认、自然打断、说话权切换和异常恢复尚未确认，不作为当前验收项。

### 6.8 气泡对话与翻译

原型还展示 AI/用户气泡消息、中英翻译、单句发音和加入错题本。这些属于完整语言练习体验，当前不作为面试主链路核心验收。

## 7. 报告、复练与语言改进模块

### 7.1 面试报告

报告包括：

- 面试用时。
- 正常完成或提前结束状态。
- 总结、优势和改进方向。
- 目标完成情况。
- 情境达成分析。
- 按实际 Turn 生成的逐题复盘卡。

每张复盘卡包含：

- AI 原问题。
- 用户完整转录。
- 原音回听。
- 证据完整或需要补充。
- 用户原话证据。
- 具体诊断。
- 下一次改进目标。
- 同题复练入口。

反馈只能基于用户真实回答，不能虚构数据或项目事实，不能冒充录用判断，也不能为了凑固定数量制造问题。

### 7.2 同题复练

```text
选择回答复盘卡
→ 查看原问题和改进目标
→ 再次回答同一道题
→ 保存新音频和转录
→ 对比原诊断缺口
→ 展示已补充和仍缺少
```

同一道题可以反复练习，每次产生独立版本，不覆盖原回答和以前的复练。

### 7.3 逐句改进

原型展示了不自然表达识别、中文解释、推荐英文表达、逐词发音和跟读练习。该能力属于后续语言提升模块，不作为当前面试主链路必验功能。

### 7.4 错题本与复习

原型展示：

- 今日待复习数量。
- 按表达、语法、词汇和发音分类。
- 查看错题来源。
- 输入优化答案。
- 查看推荐表达和解析。
- 从对话或报告加入错题本。

错题体系属于全局语言学习闭环，当前 MS1 主要验收面试专业证据复练。

## 8. 自定义角色练习模块

原型中的角色练习流程为：

```text
创建角色
→ 回答角色设定问题
→ AI 生成角色
→ 预览角色
→ 选择音色
→ 开始英语聊天
```

### 8.1 创建角色

用户可通过语音或文字设置：

- 角色类型和身份。
- 性格。
- 与用户的关系。
- 常聊话题。
- 年龄、地点和场景风格。

原型支持中英双语问题、Tip、示例以及语音/文字输入切换。

### 8.2 角色预览与音色

- 展示角色名称和身份。
- 展示中英文角色介绍。
- 试听和更换角色音色。
- 重新创建或开始聊天。

### 8.3 角色聊天

原型展示一问一答、实时通话、气泡消息、中英翻译、单句播放、Tip 和加入错题本。

角色练习是后续通用场景能力，不是当前面试 MS1 的核心真实交付。

## 9. 历史模块

历史记录的正确聚合层级为：

```text
面试计划
→ 面试官
→ 完整模拟/专项场次
→ 问答 Turn
→ 报告
→ 复练版本
```

### 9.1 历史首页

历史首页只展示计划卡，包括：

- 岗位名称。
- 示例体验标记。
- 最近练习时间和面试官。
- 总体状态。
- 继续或查看入口。

不应把每个 Session 直接堆在首页。

### 9.2 计划与面试官历史

- 计划详情展示面试官轮播、完整模拟、专项和统一历史入口。
- 面试官历史展示完整模拟和各专项的多场记录。
- 场次状态包括待开始、进行中、完成、提前结束和报告生成失败。
- 进行中的场次可以恢复。
- 已结束场次可以查看报告或再练一场。

### 9.3 场次详情

- 面试官和练习类型。
- 使用的计划、面试官及练习配置快照。
- 全部问题、回答、转录和音频。
- 报告和逐题反馈。
- 同题复练的全部版本。

## 10. 当前原型与已确认规则的冲突

| 当前原型 | 已确认产品规则 |
|---|---|
| 创建面试主要只有职位输入和简历亮点 | 三步创建：岗位、完整候选人资料、练习重点 |
| 固定生成 4 位面试官 | 系统根据岗位自动配置 2—4 位 |
| 写成固定 4 轮面试 | 实际是 2—4 位独立面试官 |
| 所有面试官共享固定专项 | 不同面试官拥有不同专项 |
| 卡片直接展示全部专项 | 完整模拟为主操作，专项放入“更多练习” |
| 按固定目标数量推进 | 按时间、问题上限和目标覆盖动态推进 |
| 可以直接切换实时对话 | 当前只预留入口，切换规则尚未确认 |
| 报告偏整体总结和语言分 | 需要按实际 Turn 生成证据复盘卡 |
| 历史显示“4 个轮次” | 首页只显示计划卡，进入后查看面试官和场次 |
| 简历原型支持 PDF、DOC、DOCX | 当前 PRD 只支持 PDF |
| 设置显示手机号 | 当前账号体系使用邮箱 |
| 权益中心和订单可见 | 当前真实版本暂时隐藏 |
| 角色练习和错题系统已可点击 | 属于终局原型，暂非面试 MS1 核心验收 |

## 11. 系统模块划分建议

整个系统可以拆成八个大的功能模块：

1. 用户、认证与数据权限。
2. 候选人资料与简历解析。
3. 练习计划与场景配置。
4. AI 角色与面试官配置。
5. 对话执行与问题控制引擎。
6. 录音、ASR、TTS 与实时语音。
7. 报告、复练与语言改进。
8. 历史、个人中心与数据管理。

## 12. 面向多场景的扩展方向

底层可以设计为通用 AI 口语练习平台，面试作为第一个场景扩展：

```text
通用练习平台
├── PracticePlan
├── AIRole
├── PracticeModule
├── PracticeSession
├── ConversationObjective
├── ConversationController
├── VoiceService
├── PracticeReport
└── PracticeHistory

场景扩展
├── InterviewScene
├── RolePlayScene
├── PresentationScene
├── MeetingScene
└── NegotiationScene
```

每种场景只需要定义自己的输入字段、AI 角色、练习模块、对话目标、控制策略、内容生成规则和报告维度，通用语音、历史、场次和复练能力可以复用。

## 13. 当前阶段结论

当前应优先完整跑通：

```text
创建面试计划
→ 自动配置面试官
→ 选择完整模拟或专项
→ 动态一问一答
→ 生成逐题报告
→ 同题复练
→ 历史查看与恢复
```

自定义角色、实时通话、气泡翻译、逐句改进和错题复习保留为通用平台的后续扩展能力，不能因为原型已经展示就默认进入当前版本验收。
