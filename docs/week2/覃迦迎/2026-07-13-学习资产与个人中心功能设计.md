# Product Killshot Report — SpeakUp E 模块

> 生成日期：2026-07-13  
> 分析范围：学习资产与个人中心  
> 原型：https://speakup-product-prototype.wendymcdonald606998.chatgpt.site/  
> 主要参照：Speak、Duolingo、ELSA Speak、Huru、Yoodli

## Executive Summary

E 模块表面上是“错题本 + 我的”，实质承担两个闭环：

1. **学习闭环**：把面试或场景对话中的问题沉淀为错题，再通过分类、作答、推荐表达和解释完成复练。
2. **个性化与商业闭环**：保存简历等长期资产，并承接账号、会员权益、订单和偏好设置。

目前最值得强调的不是“功能数量多”，而是 E 模块把一次性的 AI 对话转成了可以持续积累的个人训练系统。核心价值链是：

```text
练习中暴露问题 → 自动进入错题 → 按薄弱类型复练 → 查看可执行解析
→ 错误逐步减少 → 系统越来越了解用户
```

这条链路有明确的市面依据。Speak 会根据 Course、Free Talk 和 Roleplay 中的错误生成 Targeted Practice 与 Takeaways；Duolingo 的 Practice Tab 支持集中复习历史错误；ELSA、Huru、Yoodli证明了面试/口语产品需要结构化反馈和个性化练习。SpeakUp 的差异点不应是复制一个通用“错题本”，而是把错题绑定到**真实面试原句、岗位语境和用户简历**，形成“面试表达复练库”。

范围判断上，首期应该优先做实“错题自动沉淀—复练—解析—掌握状态”和“简历可复用”。会员、订单、头像昵称等是产品完整性与未来商业化所需，但不是当前学习价值的核心，原型占位即可，不应抢占主链路资源。

## Product Snapshot

- 产品：SpeakUp
- 定位：面向外企、出海公司和国际化岗位候选人的 AI 英文面试与口语训练工具。
- 目标用户：有一定英文基础，但在真实追问下容易卡词、表达不自然、结构混乱的技术及相关岗位求职者。
- E 模块目标：沉淀学习成果和个人背景，让用户“下次练得更准”，同时承接基础账户与商业化能力。
- 上游依赖：练习会话、逐句改进、练习报告需要产出可复练的错误项；创建模拟面试需要读取已保存简历。

## 一、目前 E 部分有什么功能

### 1. 错题回顾首页

| 项目 | 原型表现 |
|---|---|
| 使用入口 | 练习历史页顶部“今日待复习”，或左侧“错题回顾” |
| 总体状态 | 展示“待复习 6”以及预计需要 6 分钟 |
| 分类 | 表达优化、语法错误、词汇使用、发音问题 |
| 数量 | 每类展示待复习题数 |
| 最近加入 | 展示具体错误、来源练习及加入时间 |
| 核心操作 | 开始全部错题；按类型进入专项复习 |

这页解决的不是“错误存放在哪里”，而是“我今天应该先练什么”。预计时长、待复习数量和分类共同降低了复练决策成本。

### 2. 错题练习与答案解析

| 项目 | 原型表现 |
|---|---|
| 题目上下文 | 显示错题类型与来源，如“表达优化 · 产品经理面试” |
| 题目形式 | 给出用户原句，要求优化表达 |
| 听力回放 | 可播放原句 |
| 用户作答 | 文本框输入更自然的英文表达 |
| 提交状态 | “提交并查看解析” |
| 解析结果 | 推荐表达 + 具体原因说明 |
| 对比听音 | 可播放推荐表达 |
| 连续练习 | 点击“下一题”，并显示 1/6 进度 |

原型已经包含一个完整的微型学习循环：**回忆/改写 → 提交 → 对照推荐表达 → 理解原因 → 下一题**。这比单纯收藏错误更有价值，因为用户必须主动输出一次。

### 3. 加入错题

图片给出的 E 范围包含“加入错题”，而原型中的“最近加入”和错题来源也说明错误会从练习/报告环节流入错题本。当前可确认的是错题被记录并带有来源；原型未完整展示“手动加入/取消加入”的按钮状态，因此此处应作为**跨模块数据能力**而不是独立页面描述。

建议定义两种来源：

- 系统自动加入：报告判定为高价值、可复练的问题。
- 用户手动加入：用户主动收藏某一句表达或发音问题。

首期至少需要自动加入，否则错题本很容易为空；手动加入可作为补充。

### 4. 个人简历管理

| 项目 | 原型表现 |
|---|---|
| 使用入口 | “我的 → 个人简历”，或左侧快速入口 |
| 简历列表 | 展示文件名、更新时间、文件大小、格式 |
| 默认简历 | 标识默认简历，供创建面试时直接选择 |
| 上传 | 支持 PDF、DOC、DOCX，最大 10MB |
| 管理操作 | 设为默认、重命名、删除 |
| 价值说明 | AI 根据简历生成更贴合经历的问题 |

简历在这里不是普通资料附件，而是**面试个性化上下文**。它减少每次创建面试时重复上传和重复填写，也让项目经历追问更接近真实面试。

### 5. 个人中心

目前展示：

- 用户头像/昵称状态。
- 累计对话分钟数。
- 获得的激励资产（原型显示为 0 个，具体定义尚未清晰）。
- 个人简历入口及数量。
- 权益中心、会员状态与到期时间。
- 我的订单。
- 设置入口。

个人中心的作用是把“用户身份、学习资产、付费状态、系统配置”集中到一个稳定入口，不让这些低频能力打断创建和练习主流程。

### 6. 权益中心与订单

原型已展示入口、会员状态和到期时间，但点击后目前主要是提示级反馈，没有展开完整权益页和订单列表。因此汇报时应准确表述为：

> 已设计商业化入口与会员状态展示；完整购买、续费、权益明细和订单售后流程尚未在当前原型中展开。

### 7. 设置与账号信息

| 类别 | 当前功能 |
|---|---|
| 个人信息 | 头像、昵称、脱敏手机号 |
| 对话偏好 | 音色选择，当前为“亲切女声” |
| 合规信息 | 隐私政策、服务条款、关于 SpeakUp |
| 账号操作 | 退出登录、注销账号 |

设置页是账号安全、合规和偏好管理的基础设施。音色选择还直接影响长时间语音练习的舒适度。

## 二、为什么做这些功能

### 核心判断：E 模块解决“练完以后怎么办”

用户的主要痛点不是没有一次 AI 对话，而是：

- 练完后只记得“好像说得不好”，不知道具体该复练什么。
- 报告看完就结束，同类错误会在下一场面试继续出现。
- 每次创建面试都要重新提供简历和背景，个性化无法积累。
- 付费状态、账号信息和偏好没有统一管理位置。

所以 E 模块必须完成三次转化：

1. 把反馈转化为可再次作答的题目。
2. 把一次会话转化为长期个人弱项资产。
3. 把一次上传的简历转化为可复用的面试上下文。

### 各功能的产品理由

| 功能 | 为什么做 | 不做的直接后果 |
|---|---|---|
| 错题自动沉淀 | 报告只有进入后续训练，才产生长期价值 | 反馈成为一次性消费 |
| 错题分类 | 用户可以针对表达、语法、词汇、发音选择薄弱项 | 所有问题混在一起，复练无重点 |
| 今日待复习/预计时长 | 降低开始成本，把“有空再练”变成 6 分钟任务 | 错题越积越多但不打开 |
| 主动作答 | 检验用户能否重新组织表达，而不是只看懂答案 | 形成“看懂了但说不出”的错觉 |
| 推荐表达 + 解释 | 同时回答“怎么说”和“为什么这样说” | 只给标准答案，迁移到新语境的能力弱 |
| 原句/推荐表达播放 | 连接文字纠错与真实口语输出 | 错题本退化成写作练习 |
| 简历管理 | 提升追问相关性，减少重复配置 | 问题泛化，无法深挖真实项目经历 |
| 个人中心 | 集中承接身份、资产、权益和配置 | 低频功能散落，主流程变复杂 |
| 权益与订单 | 为配额、订阅和售后提供必要入口 | 后续商业化只能临时补丁 |
| 音色/账号/隐私设置 | 满足个性偏好、合规和账号安全 | 产品难以达到可正式使用状态 |

## 三、参考了市面上的什么

### 1. Speak：从错误生成针对性练习

Speak 的官方帮助中心说明，“Made For You”包含：基于课程错误的 Targeted Practice、基于 Free Talk/Roleplay 错误的 Takeaways，以及针对低熟练词汇和误用模式的练习。这与 SpeakUp 的“练习错误 → 错题分类 → 专项复练”最接近。

参考：https://help.speak.com/en/articles/11565779-what-are-made-for-you-custom-lessons

SpeakUp 应借鉴“错误自动进入下一次训练”，但不照搬泛课程体系；应绑定面试来源、岗位和简历语境。

### 2. Duolingo：把错题复习做成低门槛固定入口

Duolingo 2026 年的 Practice Tab 把 Mistakes、Words、Speak、Listen 等专项练习集中在固定入口，其中 Mistakes 用于复习历史错误。它验证了“分类专项练习 + 随时开始”是成熟学习产品的常见结构。

参考：https://blog.duolingo.com/guide-to-duolingo-practice-hub/

SpeakUp 可以借鉴入口和短任务设计，但不需要复制 XP、连胜、排行榜等游戏化外壳，因为求职用户更在意面试结果和薄弱项改善。

### 3. ELSA Speak：面试个性化与结构化教练

ELSA Interview Coach 会先收集目标岗位、行业和公司，再提供介绍动机、经历匹配、决策场景等分类练习；ELSA 也强调即时个性化反馈。这证明面试训练需要背景信息和结构化分类，而不是空白聊天。

参考：https://elsanow.freshdesk.com/en/support/solutions/articles/31000178868-coach-interview-coach

SpeakUp 的简历资产比轻量 Profile 更深入，但首期不必做复杂字段解析，只要可靠地把简历内容用于提问即可。

### 4. Huru：岗位定制问题与可执行反馈

Huru 可根据 Job Description 生成定制问题，并在反馈中分析答案相关性、表达方式、信心和 filler words，同时给出重构建议与参考答案。这支持 SpeakUp 做“来源明确的错题 + 推荐表达 + 解释”。

参考：https://huru.ai/

### 5. Yoodli：上下文追问、报告与长期进步

Yoodli 的面试教练允许用户输入目标岗位，基于回答继续追问，并在练习后生成包含语速、填充词等分析的报告，还强调长期进步追踪。这证明练习记录和反馈资产是面试教练的基础能力。

参考：https://e.yoodli.ai/use-cases/interview-preparation

## 四、为什么做这些，而不做其他功能

取舍标准不是“竞品有就做”，而是看功能是否直接服务于：

```text
更贴近真实面试的问题 → 更具体的反馈 → 更高效的复练 → 可感知的进步
```

### Must Build：应优先做实

| 优先级 | 功能 | 理由 | 工作量 |
|---|---|---|---|
| P0 | 错题自动沉淀及来源追溯 | E 模块成立的前提；来源必须能回到具体会话/句子 | 中 |
| P0 | 分类复练 + 主动作答 | 直接把反馈变成学习行为 | 中 |
| P0 | 推荐表达、解释与播放 | 让答案可理解、可模仿、可迁移 | 中 |
| P0 | 掌握状态与复习完成状态 | 当前原型只有题数，必须定义做对/待复习/已掌握 | 中 |
| P0 | 原回答与复练回答前后对比 | “看懂”不等于“会说”，对比是最短的效果证明 | 中 |
| P0 | 简历上传、默认、删除及创建面试复用 | 直接决定个性化追问质量 | 中 |
| P1 | 用户手动加入/移出错题 | 给用户控制权，补充系统自动判断 | 低 |
| P1 | 到期复习计划 | 避免错题只做一遍，可逐步引入间隔复习 | 中 |

### 保留入口，但不应成为当前重点

| 功能 | 当前建议 | 理由 |
|---|---|---|
| 权益中心 | 保留会员状态和权益说明占位 | 商业化必要，但本期核心价值尚未验证 |
| 我的订单 | 保留入口和基础订单状态 | 购买与售后基础设施，不提升首轮训练效果 |
| 头像昵称 | 做基本编辑即可 | 产品完整性功能，差异化低 |
| 音色选择 | 先提供少量稳定音色 | 有体验价值，但不应做复杂音色市场 |
| 激励资产 | 暂缓或明确含义后再做 | 原型“获得 0 个”语义不清，容易成为装饰数字 |

### 当前不做

| 功能 | 为什么不做 |
|---|---|
| 完整课程体系/课程商城 | 内容生产与运营成本高，偏离英文面试这一窄场景 |
| 排行榜、好友 PK、社区 | 求职准备具有隐私性；还存在冷启动和治理成本 |
| 复杂勋章、金币、连续打卡 | 容易奖励“打开 App”，而不是“表达问题减少” |
| 简历美化、投递、职位搜索 | 会把产品拉成大而全求职平台，与面试口语教练定位冲突 |
| 视频面试官、数字人、换装 | 演示感强但成本高，不能直接提升追问与反馈质量 |
| 音素级发音课 | 识别与内容成本高；目标是面试可理解度，不是替代 ELSA |
| 用户自建复杂题库/自定义 Agent | 高级能力，首期先验证受控题库和系统推荐质量 |
| 完整长期数据大屏 | 数据量不足时只会得到空图表；先积累可靠错题与掌握数据 |
| 多语言、多考试、多生活场景全面铺开 | 会稀释英文面试的价值主张和有限研发资源 |

## 五、E 模块页面说明与交互规则

### A. 错题回顾页

1. **模块目标**：告诉用户今天最值得复练的具体问题，并让用户快速开始。
2. **使用入口**：历史页“今日待复习”；侧边栏“错题回顾”；后续可在报告页加入“加入错题/立即复练”。
3. **页面结构**：待复习数量、预计时长、主按钮、分类卡片、最近加入。
4. **核心操作**：开始全部；选择分类；查看最近加入。
5. **状态变化**：无错题为空态；有错题为待复习态；完成后更新剩余题数和掌握状态。
6. **页面跳转**：开始/分类 → 错题练习；来源 → 对应会话详情（建议补充）。
7. **数据内容**：错题 ID、用户 ID、来源会话/轮次/原句、类型、错因、推荐表达、创建时间、复习次数、掌握状态、下次复习时间。
8. **异常情况**：无错题、音频不可用、来源会话已删除、网络失败。
9. **验收标准**：用户能在 2 次点击内从历史进入复练；分类数量与真实错题一致；完成后待复习数正确更新。

### B. 错题练习/解析页

1. **模块目标**：让用户主动改写或复述，再用对照解释完成纠偏。
2. **使用入口**：错题回顾页开始按钮或分类卡片。
3. **页面结构**：进度、类型/来源、原句、播放、作答区、提交按钮；提交后显示推荐表达、解释、推荐音频和下一题。
4. **核心操作**：播放、输入/后续可语音作答、提交、听推荐表达、下一题。
5. **状态变化**：未作答 → 可提交 → 解析展开 → 完成/下一题；提交失败保留答案。
6. **页面跳转**：下一题 → 后续错题；最后一题 → 本次复习总结（当前原型未展示，建议补充）。
7. **数据内容**：用户答案、匹配/评价结果、反馈、播放状态、完成时间、是否掌握。
8. **异常情况**：空答案、网络失败、解析失败、音频失败、重复提交。
9. **验收标准**：提交后不丢失用户答案；推荐表达与解释同时出现；最后一题完成后题数和掌握状态正确更新。

### C. 个人简历页

1. **模块目标**：复用候选人背景，让面试问题围绕真实经历展开。
2. **使用入口**：“我的 → 个人简历”；创建面试时选择简历。
3. **页面结构**：说明、简历列表、默认标识、管理菜单、上传入口。
4. **核心操作**：上传、设为默认、重命名、删除。
5. **状态变化**：上传中、解析中、可用、失败、默认、删除确认。
6. **页面跳转**：上传成功留在列表；创建面试选择后回到配置页。
7. **数据内容**：文件名、格式、大小、更新时间、解析状态、默认状态、文本内容引用。
8. **异常情况**：格式错误、超过 10MB、重复文件、无可提取文本、解析失败、网络中断。
9. **验收标准**：合法文件可上传并显示状态；只有一个默认简历；删除默认简历后有明确兜底；创建面试能读取选中简历。

### D. 个人中心/设置

1. **模块目标**：集中管理学习资产、身份、权益、订单、偏好和账号安全。
2. **使用入口**：底部导航“我的”。
3. **页面结构**：身份与统计、简历、权益、订单、设置；设置内含个人信息、音色、合规、退出/注销。
4. **核心操作**：查看/编辑、进入子页面、退出、注销。
5. **状态变化**：会员有效/即将到期/已过期；昵称未设置/已设置；登录/退出。
6. **页面跳转**：进入对应子页；注销需要二次确认。
7. **数据内容**：昵称头像、脱敏手机号、练习统计、会员期限、订单摘要、音色偏好。
8. **异常情况**：统计加载失败、会员状态不一致、订单为空、保存偏好失败。
9. **验收标准**：敏感信息脱敏；设置保存后立即生效；退出和注销含明确风险提示；权益状态与订单一致。

## 六、竞争性结论

### Quick Wins

| # | 改进 | 为什么重要 | 工作量 | 影响 |
|---|---|---|---|---|
| 1 | 在每道错题上显示“来自哪次面试、哪道问题” | 强化真实语境，区别于通用语法题库 | 低 | 高 |
| 2 | 增加“我已掌握/稍后再练” | 用户可控制复习节奏，也能形成掌握数据 | 低 | 高 |
| 3 | 最后一题后展示本次复习结果 | 当前链路缺少明确完成感 | 低 | 高 |
| 4 | 报告页增加“一键加入错题/立即复练” | 缩短反馈到行动的路径 | 低 | 高 |
| 5 | 简历显示“可用于生成问题/解析失败”状态 | 避免文件上传成功但实际不可用 | 低 | 高 |

### Switching Triggers

- 错题不是孤立单词或语法，而是用户在真实面试中说过的句子。
- 推荐表达结合目标岗位、简历项目与上下文，不是万能模板。
- 系统能证明“上次同类问题不会，这次已经能自然回答”。

### Deal-Breaker Gaps

- 错题没有可靠来源和去重，会迅速变成噪音仓库。
- 推荐表达过度标准化，用户会认为直接问通用 AI 也一样。
- 完成复练后没有掌握状态和再次安排，无法形成长期闭环。
- 简历上传成功但未真正影响问题生成，会破坏用户信任。
- 权益入口存在但规则不透明，会放大用户对自动续费的担忧。

### Lock-In Opportunities

- 个人面试表达库：按岗位、项目、行为题保存自己的最佳表达。
- 跨 Session 进步锚点：显示同类问题从“结构缺失”到“表达完整”。
- 基于面试日期和弱项的复练计划：在真正面试前安排短任务。
- 简历版本 × 目标岗位 × 历史错题的长期个性化记忆。

## 七、统一优先级

| 优先级 | 内容 |
|---|---|
| P0 现在做 | 错题来源、分类复练、主动作答、推荐表达/解释/音频、完成与掌握状态、复练前后对比、简历可用状态和复用 |
| P1 下一迭代 | 手动加入/移出、错题去重、间隔复习、复习总结、跨 Session 进步锚点 |
| P2 路线图 | 个性化表达库、面试日期计划、会员配额与完整订单流程、轻量成长趋势 |
| P3 可选 | 勋章、金币、排行榜、社区、复杂主题装扮 |

## 汇报时的一句话

> E 模块不是边缘的“我的页面”，而是负责把一次面试练习沉淀成长期学习资产：错题本让反馈可以复练，简历让下一次面试更个性化，个人中心承接资产与权益；我们参考了 Speak、Duolingo、ELSA、Huru 和 Yoodli 的成熟做法，但主动不做大课程、重游戏化和全能求职平台，因为首期最重要的是验证“真实面试错误能否被有效改掉”。
