# [提案] 模拟面试、面试官与语音练习功能范围

> 关联：[Issue #9](https://github.com/1024XEngineer/XE3-ESL/issues/9) · [最新在线原型](https://speakup-product-prototype.wendymcdonald606998.chatgpt.site/)

## 动机 / 用户故事

用户需要的不只是随机问题，而是一场有岗位背景、轮次目标、面试官职责和连续追问的训练。最新版原型已经表达“输入职位 → 生成方案 → 确认面试官 → 选择轮次/专项 → 语音练习”的完整路径，本 Proposal 需要明确其中哪些行为是产品规则，哪些只是当前产品经理样例。

典型用户故事：

- 作为一名候选人，我希望先看到并修改面试方案和面试官，再开始练习。
- 作为一名准备完整面试的用户，我希望按不同职责的面试官完成多轮模拟，感受关注点变化。
- 作为一名只想练薄弱环节的用户，我希望直接进入自我介绍、岗位八股、项目深挖、情景追问或反问环节。
- 作为一名临场表达不稳定的用户，我希望 AI 根据我刚才的回答继续追问，而不是机械切换下一题。
- 作为一名语音交互用户，我希望可以选择实时连续对话，也可以切换为主动点击说话。

一句话概括：

**模拟面试应由可确认的岗位方案驱动，以分角色轮次、受控追问和可切换语音方式形成真实训练。**

## 目标用户

- 已确定目标技术相关岗位，需要完成英文模拟面试的用户。
- 希望进行整场模拟或只重复练某个阶段的用户。
- 在连续语音和主动点击说话之间有不同环境或掌控需求的用户。

## 现有做法及其不足

| 方案 | 优点 | 主要不足 |
|---|---|---|
| 固定题库逐题回答 | 稳定、易实现 | 不使用用户背景，追问缺少因果关系 |
| 开放式 AI 聊天 | 灵活、个性化 | 容易跑题，难保证轮次和结束条件 |
| 受控面试结构 + 上下文追问 | 兼顾稳定与个性化 | 需要明确每个阶段目标和状态机 |

两个月选择第三种方案。固定题库只作为模型失败或缺少上下文时的兜底。

## 本期范围与明确不做

### 两个月最终范围

#### 创建与方案确认

- 第一步输入目标岗位，第二步编辑面试背景，第三步确认面试官。
- 系统根据已确认背景生成四轮面试计划和对应面试官。
- 用户可以编辑面试官姓名、职责和对话风格，也可以补充面试官；修改后不能破坏轮次目标。
- 当前原型中的 Mia、Ethan、Lena、Noah 仅为产品经理样例，其他岗位必须生成相应职责。

#### 四轮面试结构

四轮名称可以随岗位调整，但目的保持稳定：

| 轮次目的 | 默认关注点 | 面试官示例 |
|---|---|---|
| 综合沟通初面 | 自我介绍、动机、经历概览、基础行为问题 | HR / Recruiter |
| 岗位能力面试 | 岗位知识、判断方法、核心技能 | Hiring Manager / 岗位负责人 |
| 项目与协作深挖 | 个人贡献、取舍、验证、结果、跨职能协作 | 技术/工程/协作方负责人 |
| 业务与终面判断 | 综合案例、业务影响、优先级、反问 | 业务或高层负责人 |

- 用户可以从任一轮次选择“完整面试”。
- 用户也可以进入五类专项练习：自我介绍、岗位八股、项目深挖、情景追问、反问环节。
- “准备话术”只在训练路径提供，不在完整模拟过程中主动展示答案。

#### 语音练习

- 提供实时连续对话：系统检测用户结束说话并自动推进。
- 提供气泡/点击说话模式：由用户明确开始和结束一次回答。
- 两种模式使用同一会话上下文，切换时不丢失已完成轮次。
- 页面至少展示录音、转录、AI 思考、播放、网络异常和可重试状态。
- 文本转录是辅助信息，不以文本聊天替代核心语音体验。

#### 受控追问

- 每个阶段定义主题、核心问题、可追问目标、结束条件和反馈维度。
- AI 可以根据简历和上一轮回答进行澄清、贡献、取舍、验证、结果或收束追问。
- AI 可以调整措辞与顺序，但不能改变本阶段目标或无限延长会话。
- 缺少可靠上下文或模型失败时，使用与当前岗位/阶段匹配的兜底问题。

### Milestone 实现深度

| Milestone | 做到什么程度 |
|---|---|
| MS1 | 原型展示四位面试官、四轮计划、五类专项和两种语音方式；用一个岗位完成至少 3 轮脚本化追问 |
| MS2 | 真实跑通一个项目深挖场景：真实语音/转录、至少 2 次受控追问和基础报告；面试官可使用固定模板 |
| MS3 | 交付岗位自适应四轮面试、五类专项、面试官编辑/补充、两种语音方式和历史恢复 |
| MS4 | 完成超时、打断、弱网、识别失败、模型失败、恢复、稳定性和跨岗位测试 |

### 明确不做

- 不做多名面试官同时在线的群面、视频会议或实时多人协作。
- 不做数字人、口型、3D 形象或复杂面部动画。
- 不允许面试官成为具有任意工具调用能力的通用 Agent。
- 不做无限制自由出题；问题必须服从当前岗位和阶段目标。
- 不在完整模拟中逐句打断纠错或展示完整参考答案。
- 不以“面试官数量越多”作为价值，默认四个职责已足够表达主要评价视角。

## 关键决策与依据

### 决策 1：默认四个职责视角，按轮次依次出现

四个视角能覆盖综合沟通、岗位能力、项目协作和终面判断。它们代表评价职责，不要求四个真实 AI 同时运行。按轮次依次出现既符合用户认知，也显著降低多人会话状态复杂度。

### 决策 2：轮次目标稳定，内容随岗位变化

产品经理样例中的“研发协作交付”不能直接复制到所有岗位；后端工程师、测试、算法等岗位应生成与其职责一致的轮次描述和问题。保持稳定的是轮次目的与追问结构，而不是页面上的具体文案。

### 决策 3：完整面试与专项训练共用内容，不维护两套题库

- 完整面试：强调连续性和临场压力，默认不显示提示，不允许任意回退。
- 专项练习：允许准备话术、最小提示、暂停和重答。

两者使用相同岗位背景、面试官和阶段目标，避免内容与报告标准分裂。

### 决策 4：实时与点击说话两种交互并存

实时模式更接近自然对话，点击说话模式在嘈杂环境、演示和识别不稳定时更可控。两种模式共享会话状态，避免被实现成两个相互割裂的产品流程。

### 决策 5：受控材料 + AI 动态追问

示例因果链：

```text
Q1：说明一个你做过的重要项目或决策。
Q2：基于回答中提到的方案，追问考虑过哪些替代方案以及为什么选择它。
Q3：继续追问如何验证，以及产生了什么可核对的结果。
```

示例不绑定具体岗位；追问必须引用用户刚才提到的信息，不能只是预先排好的下一题。

## 基本概念与信息结构

| 概念 | 定义 |
|---|---|
| Interview Plan | 根据本次面试背景生成并由用户确认的四轮安排 |
| Interviewer | 对应一轮面试的职责、关注点、姓名和对话风格 |
| Interview Round | 完整面试中的一个主题轮次 |
| Practice Stage | 可单独重复练习的五类面试阶段之一 |
| Practice Mode | 完整模拟或专项训练 |
| Voice Mode | 实时连续对话或气泡/点击说话 |
| Session | 用户一次连续练习及其当前状态 |
| Turn | 一个问题/追问与一个用户回答 |
| Follow-up Goal | 澄清、贡献、取舍、验证、结果或收束 |

```text
Draft Context
  -> Generated Plan
  -> Confirmed Interviewers
  -> Select Round / Stage
  -> Session Started
  -> AI Question
  -> User Voice Answer
  -> Follow-up or Advance
  -> Session Completed
  -> Report Generated
```

语音协议、模型编排、状态存储、超时和上下文窗口属于架构 Proposal。

- “准备话术”只属于专项训练。
- “实时对话”与“气泡/点击说话”两种模式均进入最终范围。
- MS1 允许问题、语音和状态使用 Mock，但至少一次追问要让评审看出它依赖上一轮回答。

## 验收标准

### Proposal 定稿验收

- [ ] 团队确认三步创建、四个职责视角、四轮结构和五类专项。
- [ ] 团队确认面试官按轮次出现，不做多人同时面试。
- [ ] 团队确认产品经理、面试官姓名和轮次文案均为可替换样例。
- [ ] 团队确认完整模拟与专项训练的提示规则不同，但共用同一内容模型。
- [ ] 团队确认实时和点击说话两种交互都进入两个月最终范围。
- [ ] 团队确认采用受控阶段目标 + 上下文追问，不做无限出题。

### 两个月产品行为验收

1. 用户输入任一已支持技术相关岗位后，面试官职责、轮次描述和问题都与该岗位一致。
2. 用户可以确认或修改面试官，并进入完整面试或任一专项练习。
3. 项目深挖会话至少包含 3 个用户回答和 2 次依赖上下文的追问。
4. 训练模式可以查看准备话术或最小提示，完整面试默认不展示答案。
5. 用户可在实时与点击说话模式之间切换，已完成轮次不会丢失。
6. AI 偏离主题时能收束；达到阶段目标或结束条件时会推进或结束。
7. 录音、识别、播放、模型或网络失败时有明确状态和重试入口。
8. 用户可以从历史恢复允许恢复的会话并查看已完成转录。
