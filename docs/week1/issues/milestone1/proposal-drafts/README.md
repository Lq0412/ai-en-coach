## 本轮要确定的共同结论

- 产品定位：英文技术面试教练，不扩展为通用英语口语平台。
- 原型职责：表达未来两个月的最终产品形态，不代表 MS1 要实现所有页面和能力。
- 目标岗位：由用户输入，首期覆盖研发、测试、数据/算法、运维/DevOps、技术产品等技术相关岗位；任何具体岗位都只能作为可替换的演示样例。
- MS1 主链路：选择一个代表性技术岗位的“项目经历深挖”作为完整演示场景，跑通背景设置、三轮追问、证据化反馈和同题复练。
- 用户模块：两个月内采用最小账户体系；MS1 只使用演示用户，不实现完整登录。
- 简历模块：简历可选；支持 PDF，失败或不上传时使用目标岗位、经验年限和目标公司类型兜底。
- 产品差异化：基于职位、简历和上一轮回答继续追问；反馈引用用户原回答；反馈后可以直接重答并比较改进。
- 明确排除：通用英语场景、会员支付、积分排行、社交、数字人、多面试官角色、音素级评分、简历编辑/ATS 打分、自定义 Agent。

## 当前原型需要在 Proposal 通过后对齐的点

- 保留：创建面试、四轮面试计划、专项练习、语音对话、历史、报告、复练、用户与简历管理。
- 补充：行为面试在完整面试中的归属；无简历时的三项快速背景；简历解析失败状态；复练前后对比状态。
- 收敛：简历格式统一为 PDF；用户页不突出手机号、头像、昵称等非核心账户能力。
- 删除或不再扩展：会员、订单、积分、排行榜、通用英语、自定义角色、数字人、好句收藏库。
