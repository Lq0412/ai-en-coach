# 设计规范：ChatGPT 极简黑白风格（Design System）

> **用途**：这是一份供 AI Agent / 设计师复用的设计语言文档。目标不是像素级克隆 ChatGPT，而是让另一个 Agent 仅凭本文即可复现同样的**克制、单色、高留白**的产品气质。
>
> **证据来源与边界**：本文基于一张 ChatGPT 移动端「项目」空状态截图提炼。
> - **观测**（截图直接可见）：配色关系、形状、布局、层级。
> - **推断**（标注 `【推断】`）：精确 hex 值、暗色模式、hover/active 等交互态、字体族——单张截图无法确证，落地时请以真机为准校准。

---

## 1. 视觉主题与气质（Visual Theme & Atmosphere）

- **整体感觉**：安静、克制、类纸张。近乎纯白的画布上，只有黑与灰在工作，没有任何彩色装饰。信息靠**留白和字重**分层，而不是靠颜色和边框。
- **视觉密度**：极低。大段留白，元素之间距离宽松，空状态几乎占满整屏，中心化排布。
- **品牌姿态**：工具型、专业、成人化。不卖萌、不炫技，让内容和操作本身说话。
- **标志性母题**：
  - **纯黑 pill 胶囊按钮**作为唯一的强操作信号
  - **浅灰圆形/圆角容器**承载图标与次级控件
  - **单色 outline 线性图标**，无填充、无渐变
  - 大字重标题 + 灰色副文案的经典双行结构

### 关键特征（Key Characteristics）

- **单色系统**：黑 / 白 / 灰三档，无品牌彩色、无渐变、无阴影堆叠。
- **形状语言统一**：强按钮=全圆角胶囊；容器=中等圆角方；图标钮=正圆。
- **留白即层级**：靠间距而非分割线组织信息。
- **一个焦点**：每屏只有一个纯黑主操作，视线无需犹豫。

## 2. 色彩系统与角色（Color Palette & Roles）

> 采用语义命名。hex 值为 `【推断】`（基于 ChatGPT 已知配色 + 截图观察），落地时请以真机取色校准。

| 角色 | 语义名 | 值（推断） | 用法 |
| --- | --- | --- | --- |
| 主操作 / 主文字 | `ink` | `#0D0D0D` | 主 CTA 填充、标题、正文主色 |
| 画布背景 | `canvas` | `#FFFFFF` | 页面底色 |
| 次级表面 | `surface` | `#F4F4F4` | 搜索框、图标容器、选中态 pill 底 |
| 更浅表面 | `surface-soft` | `#F7F7F8` | 卡片/分区的极浅底 |
| 次要文字 | `text-secondary` | `#8E8E93` | 副标题、说明文案、占位符 |
| 未选中文字 | `text-muted` | `#9B9B9B` | 未激活的 Tab、弱化标签 |
| 描边 | `border` | `#E5E5E5` | 分隔线、输入框边（若需要，克制使用） |
| 反白 | `on-ink` | `#FFFFFF` | 黑底按钮上的文字与图标 |

### Primary（主色）

- **`ink #0D0D0D`**：唯一的"重"色。承担两个职责——① 全部正文/标题文字；② 主操作按钮（如「新项目」）的胶囊填充。**页面上任何纯黑块 = 可点的主操作。**
- 白 `#FFFFFF`：画布，负责制造呼吸感。

### Interactive（交互色）

- 主操作按钮：黑底白字胶囊。
- **【推断】** 无独立的链接蓝 / focus 高亮色——整个系统避免使用彩色，交互反馈依赖明度变化（如按下时轻微变暗）与背景灰块的出现/消失。

### Neutral Scale（中性阶）

- `#0D0D0D` — 主文字 / 主按钮
- `#8E8E93` — 次要文字
- `#9B9B9B` — 未激活 Tab
- `#E5E5E5` — 描边
- `#F4F4F4` — 次级表面
- `#F7F7F8` — 更浅表面
- `#FFFFFF` — 画布

### Surface & Overlay（表面与浮层）

- 次级控件（返回钮、加号钮、搜索框、图标容器）统一使用 **浅灰填充 `#F4F4F4`**，不描边——靠底色与画布的明度差自然浮起。
- 选中态 Tab「全部」使用同一浅灰胶囊底 + 深色文字来表达激活。

### Theme Modes（主题模式）

#### Light Mode（浅色 · 已观测）

- Background: `#FFFFFF`
- Surface: `#F4F4F4` / `#F7F7F8`
- Text: `#0D0D0D`（主）/ `#8E8E93`（次）
- Accent: 无彩色，`#0D0D0D` 充当强调
- Notes: 截图即此模式。

#### Dark Mode（深色 · 【推断】未观测）

- **【推断】** ChatGPT 有深色模式，但本次截图未包含。若需支持，典型映射为：Background `#212121`、Surface `#2F2F2F`、Text `#ECECEC`（主）/ `#B4B4B4`（次），主按钮反转为**白底黑字胶囊**。
- 落地前请务必以真机深色模式取色，勿直接采用上列推断值。

### Shadows & Depth（阴影与深度）

- **几乎不用阴影。** 层级靠"浅灰块浮在白底上"来表达，而非投影。
- **【推断】** 如需极轻提升（如弹出菜单），使用单层柔和阴影即可，避免多层堆叠。

## 3. 字体规范（Typography Rules）

### Font Family（字体族）

- Primary: **【推断】** 系统无衬线体——iOS 上为 `SF Pro Text`，Android 为 `Roboto`，中文回退 `PingFang SC` / `Microsoft YaHei`。ChatGPT 近期也在用其自有 sans，但移动端视觉上等同系统体。
- Monospace: 代码/等宽场景 **【推断】** `SF Mono` / `ui-monospace`。
- 建议 CSS 栈：`"SF Pro Text","PingFang SC","Microsoft YaHei",system-ui,-apple-system,"Segoe UI",Roboto,sans-serif`

### Hierarchy（层级）

> 尺寸为基于截图比例的 `【推断】` 估值，用作相对关系参考。

| 角色 | 字号 | 字重 | 说明 |
| --- | --- | --- | --- |
| 页面标题（导航栏中央「项目」） | ~18px | 600 | 居中，深色 |
| 空状态大标题（「开始你的第一个项目」） | ~22px | 700 | 深色，居中，字重是主要强调手段 |
| 副说明文案 | ~15px | 400 | 次要灰 `#8E8E93`，居中 |
| 主按钮文字（「新项目」） | ~17px | 600 | 反白，胶囊内 |
| Tab 文字 | ~16px | 选中 600 / 未选 400 | 选中深色，未选灰 |
| 占位符（「搜索」） | ~16px | 400 | 次要灰 |

### Principles（排版原则）

- **字重承担强调**：从灰到黑、从 400 到 700 的跳变，替代了彩色高亮的作用。
- **两行结构**：`大字重标题 + 灰色说明`是全站复用的信息单元。
- **居中用于空/引导态**，列表/内容态则左对齐（**【推断】**，截图为空状态）。
- 行高宽松，不拥挤。

## 4. 组件样式（Component Stylings）

### Buttons and Links（按钮与链接）

- **主 CTA**：纯黑 `#0D0D0D` **全圆角胶囊**（`border-radius: 999px`），白字，水平内边距充足（视觉上约 `14px × 40px+`），居中。全屏唯一，不与其他强色竞争。
- **图标钮（返回 / 加号）**：**浅灰圆形** `#F4F4F4` 底，直径约 44px，内含深色 outline 图标，无描边。
- **【推断】** 次级按钮：浅灰胶囊 + 深色文字（与选中 Tab 同款）。
- **【推断】** Hover/Active：按下时底色轻微变暗（如黑按钮 → `#000`，灰钮 → `#E8E8E8`），无位移、无彩色反馈。

### Cards and Containers（卡片与容器）

- 表面风格：浅灰填充，**无边框或极浅边框**。
- 圆角：中等（图标容器约 `12–16px`）。
- 阴影：无 / 极轻。
- 内边距：宽松。
- 空状态图标容器：浅灰 `#F4F4F4` 圆角方块，居中承载单色 outline 图标。

### Inputs and Interactive Controls（输入与控件）

- **搜索框**：浅灰 `#F4F4F4` **全圆角胶囊**，左侧放大镜 outline 图标 + 灰色占位符「搜索」，无可见描边。固定在屏幕底部。
- **【推断】** Focus：出现细边框或底色略深，无彩色 focus ring。

### Navigation（导航）

- 顶部导航栏：`左返回钮（灰圆） — 居中标题 — 右加号钮（灰圆）` 三段式。
- 背景：与画布同为纯白，无分割线。
- **Tab 组**：文字型分段，`选中项 = 浅灰胶囊底 + 深色文字`，`未选项 = 无底 + 灰色文字`。无下划线指示器。

### Image / Icon Treatment（图标处理）

- **统一线性 outline 图标**：`stroke` 描边、无填充、圆角线帽，`stroke-width` 约 1.8–2，颜色随文字（深色或灰）。
- 无彩色图标、无 emoji 化、无拟物、无渐变。

### Distinctive Components（标志性组件）

- **黑色胶囊主按钮**：产品的视觉锚点。
- **灰圆图标钮**：所有次级操作的统一外壳。
- **空状态三件套**：灰圆角图标容器 + 黑色大标题 + 灰副文案 + 黑胶囊 CTA，垂直居中。

## 5. 布局原则（Layout Principles）

### Spacing System（间距系统）

- 基础单位：**【推断】** 8px 网格（4/8/12/16/24/32…）。
- 特征：**大间距**。区块间距远大于常规，制造"空"的高级感。

### Grid & Container（栅格与容器）

- 单列为主，移动端全宽 + 左右安全边距（约 20–24px）。
- 空/引导态内容**垂直水平双居中**。

### Whitespace Philosophy（留白哲学）

- **留白是一等公民**。宁可空，不填满。
- 用间距而非分割线/边框来划分区域。
- 内容态左对齐，引导态居中。

### Border Radius Scale（圆角阶）

- 微：~8px（**【推断】** 小标签）
- 标准：12–16px（图标容器、卡片）
- 胶囊：`999px`（主按钮、搜索框、选中 Tab）——**这是最标志性的形状**。

## 6. 深度与层次（Depth & Elevation）

| 层级 | 处理 | 用途 |
| --- | --- | --- |
| 平面 | 纯白画布 | 页面底 |
| 浅灰浮起 | `#F4F4F4` 色块，无阴影 | 控件、容器、选中态 |
| 主操作 | 纯黑胶囊 | 唯一强操作 |
| 浮层【推断】 | 单层柔和阴影 | 菜单/弹窗 |

### 深度原则

- **明度即层级**：越靠前的元素不一定越"高"，而是明度对比越强（黑按钮 > 灰块 > 白底）。
- 阴影语言：几乎为零，避免拟物化。
- 不使用玻璃拟态 / 大面积模糊。
- 仅在真正的浮层（菜单）才引入极轻阴影。

## 7. 该做与不该做（Do's and Don'ts）

### Do（该做）

- 用**黑白灰三档**完成全部界面。
- 每屏保留**唯一一个纯黑主操作**。
- 用**留白和字重**建立层级。
- 图标统一为**单色线性 outline**。
- 强按钮和搜索框用**全圆角胶囊**。

### Don't（不该做）

- ❌ 引入品牌彩色、渐变、发光、霓虹。
- ❌ 给每个卡片加边框 + 阴影 + 底色（"卡片套娃"）。
- ❌ 一屏出现多个争抢注意力的强按钮。
- ❌ 用填充式/彩色/emoji 图标破坏单色系统。
- ❌ 用分割线密集切割界面——优先用间距。

## 8. 响应式行为（Responsive Behavior）

> 截图为移动端竖屏（约 390–414px 宽）。以下为 `【推断】`。

### Breakpoints（断点）

| 名称 | 宽度 | 关键变化 |
| --- | --- | --- |
| Mobile | ~390px | 单列、底部固定搜索、居中空状态（已观测） |
| Tablet | ~768px | **【推断】** 内容居中限宽，两侧留白增大 |
| Desktop | ~1024px+ | **【推断】** 可能出现侧栏，主区限宽居中 |

### Touch Targets（触控目标）

- 图标钮 ≥ 44×44px（已符合）。
- 主胶囊按钮高度 ≥ 48px，易点。

### Collapsing Strategy（收拢策略）

- **【推断】** 移动端优先单列；宽屏时不铺满，而是**限制内容最大宽度并居中**，维持留白气质。

## 9. Agent 使用指引（Agent Prompt Guide）

### 快速色彩参考（Quick Color Reference）

- 主操作 / 主文字：`#0D0D0D`
- 背景：`#FFFFFF`
- 标题文字：`#0D0D0D`（字重 700）
- 正文/副文案：`#8E8E93`
- 次级表面（搜索框/图标钮/选中态）：`#F4F4F4`
- 描边（克制使用）：`#E5E5E5`
- 强调色：无——用黑与字重代替

### 一句话总结（Quick Summary）

> ChatGPT 极简风：纯白画布 + 黑白灰三档单色系统，零彩色零渐变零阴影。强操作永远是**纯黑全圆角胶囊**且每屏唯一；次级控件是**浅灰圆形/胶囊**无描边。图标全为**单色线性 outline**。层级靠**留白与字重**而非颜色和边框。整体安静、专业、类纸张，一屏只给一个焦点。

### 组件 Prompt 示例（可直接粘贴）

- **主按钮**：`做一个纯黑 #0D0D0D 全圆角胶囊按钮（border-radius:999px），白色文字 600 字重，水平内边距充足，页面唯一强操作。按下时底色变纯黑，无位移无彩色反馈。`
- **图标钮**：`44×44px 正圆，浅灰 #F4F4F4 填充无描边，内含深色单色线性 outline 图标（stroke-width 约 2）。`
- **搜索框**：`全圆角胶囊 #F4F4F4 底无描边，左侧放大镜 outline 图标 + 灰色 #8E8E93 占位符文字，固定屏幕底部。`
- **空状态**：`垂直水平居中：浅灰 #F4F4F4 圆角方块内放单色 outline 图标 → 黑色 22px/700 大标题 → 灰色 15px 说明文案 → 黑色胶囊 CTA。全程无彩色、无边框、大留白。`

### 可直接使用的 Prompt（Ready-to-Use）

> 用 ChatGPT 极简黑白风重做这个界面：纯白背景，只用黑（#0D0D0D）白灰（#F4F4F4/#8E8E93）三档，禁止任何彩色/渐变/发光/阴影。主操作用纯黑全圆角胶囊按钮且每屏唯一；返回、加号等次级操作用浅灰圆形图标钮无描边；搜索框用浅灰胶囊。所有图标为单色线性 outline。用大留白和字重（400/600/700）建立层级，不用分割线堆砌。整体安静、专业、类纸张。

### 迭代指引（Iteration Guide）

1. **先减后加**：默认全灰白，只在唯一主操作上"点"一处黑。
2. **查彩色泄漏**：任何非黑白灰的颜色都要有充分理由，否则删掉。
3. **查形状一致**：强按钮/搜索/选中态是否都用了胶囊；图标钮是否都是正圆。
