## 一、需求背景与选型标准

### 1.1 业务场景

AI 英语口语教练是一个实时对话式教学产品，核心流程为：

> **用户语音输入 → STT 识别 → LLM 生成回复 → TTS 合成语音 → 播放给用户**

TTS 环节直接决定用户的**听感体验**和**学习效果**——如果发音不标准、语调机械，将直接影响学习者的语音模仿质量。

### 1.2 选型核心指标

针对英语口语教学场景，TTS 选型需满足以下五个维度的特殊要求：

| 维度           | 要求                                          |  权重   |
| ------------ | ------------------------------------------- | :---: |
| **语音质量**     | 英语发音必须达到母语级别，包括语调、韵律、连读、重音等细节               | ⭐⭐⭐⭐⭐ |
| **实时延迟**     | 对话场景要求 < 300ms 首音频时间，> 500ms 会破坏对话节奏        | ⭐⭐⭐⭐⭐ |
| **表现力控制**    | 需支持语速调节（0.5x-1.5x）、情感表达（鼓励/热情/耐心），适配不同水平学习者 | ⭐⭐⭐⭐  |
| **经济性**      | 口语练习多为高频使用场景，需控制单次对话成本，支持规模化                | ⭐⭐⭐⭐  |
| **稳定性与可运维性** | 声音一致性、服务 SLA、接入复杂度、国内网络可达性                  |  ⭐⭐⭐  |

### 1.3 国内场景特殊考量

- **网络可达性**：所有方案均在国内可直接访问，无需代理
- **数据合规**：国内厂商数据存储符合中国法规要求
- **中文支持**：所有方案对中文有天然优势，中英混合场景适配好
- **部署灵活性**：部分方案支持私有化部署和离线 SDK

## 八、风险与注意事项

### 8.1 各方案关键风险

| 方案 | 风险点 | 缓解措施 |
|------|--------|---------|
| 科大讯飞 | 按次计费，长文本不划算 | 长文本场景切换按字符计费方案 |
| 腾讯云 TTS | 免费额度仅 3 个月 | 提前规划预付费资源包 |
| 百度智能云 | 精品版成本高 | 控制精品版使用比例 |
| CosyVoice | API 成熟度不如大厂 | 做好降级和 fallback 方案 |
| 火山 megaTTS | 音色授权费隐藏成本 | 提前确认所需音色是否为基础音色 |
| MiniMax | 中文字符双倍计费 | 精确测算实际成本 |
| Xiaomi MiMo | 限时免费不可持续，正式价格和 SLA 未明确 | MVP 先用，生产前同步验证 MiniMax / CosyVoice 备用 |
| 有道智云 | 按次计费不适合长文本 | 仅用于单词/短句场景 |
| 魔音工坊/标贝 | API 定价不透明 | 提前与商务沟通获取明确报价 |
| 开源方案 | 运维成本、GPU 稳定性 | 使用云端 GPU 按量服务降低运维压力 |

### 8.2 通用建议

1. **始终准备降级方案**：至少配置两个 TTS 供应商，主备切换（如腾讯云 + CosyVoice）
2. **音频缓存策略**：对高频固定话术（如"你好，让我们开始吧"）进行缓存，减少 API 调用
3. **数据安全合规**：如涉及未成年用户，确保 TTS 供应商符合数据保护法规
4. **监控与告警**：建立 TTS 质量监控（MOS 评分）、延迟监控和成本监控
5. **AB 测试**：上线前对不同方案进行盲听 AB 测试，以真实用户反馈为最终标准
