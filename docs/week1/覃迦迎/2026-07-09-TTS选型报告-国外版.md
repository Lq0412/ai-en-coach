## 一、需求背景与选型标准

### 1.1 业务场景

AI 英语口语教练是一个实时对话式教学产品，核心流程为：

> **用户语音输入 → STT 识别 → LLM 生成回复 → TTS 合成语音 → 播放给用户**

TTS 环节直接决定用户的**听感体验**和**学习效果**——如果发音不标准、语调机械，将直接影响学习者的语音模仿质量。

### 1.2 选型核心指标

针对英语口语教学场景，TTS 选型需满足以下五个维度的特殊要求：

| 维度 | 要求 | 权重 |
|------|------|:----:|
| **语音质量** | 英语发音必须达到母语级别，包括语调、韵律、连读、重音等细节 | ⭐⭐⭐⭐⭐ |
| **实时延迟** | 对话场景要求 < 300ms 首音频时间，> 500ms 会破坏对话节奏 | ⭐⭐⭐⭐⭐ |
| **表现力控制** | 需支持语速调节（0.5x-1.5x）、情感表达（鼓励/热情/耐心），适配不同水平学习者 | ⭐⭐⭐⭐ |
| **经济性** | 口语练习多为高频使用场景，需控制单次对话成本，支持规模化 | ⭐⭐⭐⭐ |
| **稳定性与可运维性** | 声音一致性、服务 SLA、接入复杂度 | ⭐⭐⭐ |

### 1.3 国外方案特殊考量

- **英语发音优势**：国际厂商在英语自然度和发音准确性上普遍优于国内方案
- **网络可达性**：大部分方案在国内需代理访问，增加运维复杂度
- **数据合规**：数据存储在海外，需关注国内数据保护法规
- **计费方式**：多以美元计价，存在汇率波动风险

### 8.2 通用建议

1. **始终准备降级方案**：至少配置两个 TTS 供应商，主备切换
2. **代理部署考量**：如需代理，评估代理稳定性、延迟增加和额外成本
3. **音频缓存策略**：对高频固定话术进行缓存，减少 API 调用
4. **数据安全合规**：海外方案数据存储在海外，需关注国内数据保护法规
5. **汇率风险**：美元计价的方案需考虑汇率波动对成本的长期影响
6. **AB 测试**：上线前对不同方案进行盲听 AB 测试，以真实用户反馈为最终标准
