# Idea 2：把“豆包 + DeepSeek 自制外教”从教程变成可持续训练产品

> 一句话主张：用户已经在用 DeepSeek 生成提示词、用豆包打语音电话；我们不重复造聊天入口，而是把复杂配置、学习记忆、复习和进步证明封装成一个开箱即用的个性化口语教练。

## 为什么值得验证

这个方向同时具备大平台流量和明确的用户自发用法。QuestMobile 披露，截至 2026 年 3 月，豆包和 DeepSeek 月活分别为 3.45 亿和 1.27 亿；豆包月人均使用 54.8 次，DeepSeek 为 41.7 次。大用户量不能直接等于口语学习需求，但说明教育用户使用这两个工具的成本很低。

Agent Reach 在 2026 年 7 月 8 日搜索小红书“DeepSeek 豆包 英语口语”“豆包 英语口语 陪练”，发现：

高收藏通常意味着用户想照做，但评论也暴露了阻力：反复有人问“怎么选语言”“为什么没有创建智能体”“能否分享提示词”。另一篇评论直接索要口令，并询问如何设置翻译和纠正发音、语法。说明当前方案**被验证的是教程需求，不是完整产品留存**：配置仍复杂，效果依赖提示词，而且聊天后缺少系统复习。

## 我们为谁解决什么问题

优先人群是价格敏感、已经知道豆包或 DeepSeek 的中国英语学习者：

- 大学生、初入职场者和下沉城市用户，希望免费或低价获得高频练习；
- 英语大致 A1—B1，不知道如何写 prompt，也不知道该练什么；
- 有具体目标，如求职面试、出国生活、工作会议或日常开口，希望内容与自己相关。

他们现在要自己完成“描述水平 → 设计角色 → 写指令 → 配置豆包 → 判断反馈是否合理”。即使成功，通用 Agent 通常也不知道哪些表达曾经卡住、何时该复习，以及用户是否真的从“看懂答案”变成“能自己说”。

## 我们做什么

产品形态不应只是售卖一份万能提示词。更合理的首版是一个 **Agent 配置器 + 轻量学习后台**：

1. 用 3 分钟中文问答获取目标、水平、兴趣、可用时间和纠错偏好；
2. 自动生成可导入豆包的专属 Agent 配置，DeepSeek 负责规划和改写，豆包承载熟悉的语音入口；
3. 每次会话只提取 1—3 个真正阻碍表达的卡点，而不是罗列所有错误；
4. 下次自动生成与旧卡点相关、但不重复答案的新情境；
5. 用“上次需要完整句 → 这次只需关键词 → 本次独立说出”展示进步。

真正的产品资产不是某个提示词，而是用户的 **学习状态层**：目标、真实生活话题、表达卡点、支援等级和跨场景激活记录。未来是否继续依赖豆包可以再决定，首轮先借现成基础设施验证需求。

## 风险与边界

- 豆包的 Agent 创建入口、角色、声音或开放能力可能变化；已有评论出现功能入口找不到，平台依赖是真实风险。
- “DeepSeek + 豆包”容易被复制，若只有提示词，没有学习记录和效果闭环，就不是可防守产品。
- 不能把两款 App 的总月活当作口语市场规模；真正要测的是目标人群的使用频次和付费意愿。
- 若会话数据无法可靠导出，就先通过用户上传转写、分享摘要或独立网页记录验证，不承诺尚未打通的集成。

## 最小验证

先做可人工交付的测试：招募 30 名用户，分为“现成通用提示词”和“个性化配置 + 跨天卡点复现”两组，持续 14 天。

核心指标：配置完成率；首周完成 3 次语音练习的比例；第二周仍练习的比例；无提示激活的表达数量；用户是否愿意为自动记录与复习支付 19—39 元/月。若个性化只提高首次新鲜感，却没有提高第二周留存或表达激活率，应停止把它包装成独立产品。

## 结论

这个 Idea 的优势是触达快、成本低、教育成本小，适合先验证中国大众用户。它的弱点同样明确：平台依赖重、提示词没有壁垒。因此应把“豆包 + DeepSeek”当作验证和分发方案，把长期价值放在个性化学习记忆与可验证进步上。
