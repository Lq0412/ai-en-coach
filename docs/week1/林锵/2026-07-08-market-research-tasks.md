## 一、本周总目标

在确定产品方向与开发方案之前，先用证据回答 **4 个问题**：

1. **谁**在练口语？谁最急、最愿付费？
2. **他们现在用什么**？为什么不满意？
3. **市场空白在哪**？头部竞品没做好什么？
4. **我们 2 个月能验证什么**？（用户可感知的 Demo 层面，不涉及技术选型）

### 周五评审交付物

## 三、T1 — 用户需求调研（访谈 + 社媒）

**目标**：用 **真人对话（深度）** + **公开社区（广度）** 两条线，收集「谁在痛、痛什么、现有 App 为什么不够」。

**与假设 H1 的关系**：7.7 文档中的「冲突 / 高压口语」方向作为 **假设 H1** 嵌入本任务——访谈和社媒中 **单独标记**，但不预设为最终方向。详见 [第八节](#八假设-h1冲突高压口语方向的处理)。

### 3.1 子任务 A：用户访谈

#### 数量与人群

- 完成 **6–8 次**，每次 **15–20 分钟**（语音 / 视频 / 微信均可）
- 人群尽量分散，**不要** 8 个都是熟人或同一类型

| 类型 | 至少人数 | 目的 |
|------|----------|------|
| 外企 / 互联网职场 | 2 | 会议、接话、邮件口语 |
| 留学 / 准备出国 | 2 | 日常、学术场景 |
| 海外生活 / 曾长期 abroad | 1–2 | 维权、客服、生活英语 |
| 考试向（雅思 / 托福等） | 1 | 模考压力 |
| 用过 Speak / 流利说 / 扇贝等 | ≥3 | 切换或放弃原因 |

#### 访谈提纲（固定问题）

**开场**：你最近一次「需要用英语说出来」是什么情况？

1. 当时卡在哪？（不敢说 / 脑子空白 / 发音 / 词不够 / 不知道对不对）
2. 你后来怎么解决的？用过 App 或别的办法吗？
3. 如果有一个工具，只能帮你 **一件事**，你希望是什么？
4. （若有过争执 / 拒绝 / 面试 / 维权）如果提前练 10 分钟，你觉得有用吗？
5. 这类事你 **多久遇到一次**？愿意付费吗？什么形式 / 什么价位？

**假设 H1 追加**（仅当对方聊到过高压场景时问）：

- 现有口语 App 练的内容，和真实场景 **像不像**？
- 你需要的是「更礼貌日常」还是「更硬气、能把事说清楚」？

#### 交付物

| 文件 | 路径建议 | 要求 |
|------|----------|------|
| 访谈纪要 | `research/T1-用户调研/interviews/` | 每人 1 个 md：背景标签 + 5 问摘要 + **≥2 句用户原话** |
| 场景地图 | `research/T1-用户调研/场景地图.csv` | ≥12 行，字段见 [附录 A](#附录-a场景地图csv-字段) |

**验收**：≥6 份纪要；场景地图满字段；原话可直接引用。

### 3.2 子任务 B：社媒调研（小红书等）

#### 平台与关键词

每个平台 **20–30 条有效内容**（帖子 / 评论 / 高赞回答），**必须带链接**。

| 平台 | 建议搜索关键词（组合使用） |
|------|---------------------------|
| **小红书** | 英语口语 App、不敢开口英语、外企英语、雅思口语、海外维权 英语、Speak 测评、流利说 口语 |
| **知乎** | 英语口语怎么练、Speak 怎么样、外企 开会 英语、留学 不敢说话 |
| **B 站** | 口语 App 测评、雅思口语、职场英语（**重点看评论区**） |
| **豆瓣**（可选） | 留学 / 外企相关小组内「口语」帖 |

#### 采集原则

1. **优先** 真实求助、吐槽、经验帖；**少算** 营销号「Top10 App 推荐」
2. 每条必须写 **具体场景**（面试 / 开会 / 维权…），不要只写「想练口语」
3. 访谈与社媒 **交叉验证**：两边都高频 → 周五矩阵加粗；仅一边有 → 标「待补证」

#### 交付物

| 文件 | 路径建议 | 要求 |
|------|----------|------|
| 原始摘录表 | `research/T1-用户调研/社媒/raw-摘录.csv` | **≥80 条**（四平台合计），字段见 [附录 B](#附录-b社媒摘录csv-字段) |
| 标签统计 | `research/T1-用户调研/社媒/痛点标签统计.md` | 标签 \| 条数 \| 代表链接 ×2 |
| Top10 需求 | `research/T1-用户调研/社媒/Top10需求.md` | 按频次排序 + 每条 1 句用户原话 |
| H1 专项统计 | 合并在 `用户需求证据摘要.md` | 「高压 / 不敢争 / App 太温和」出现次数 + 典型原话 5 条 |

**验收**：四平台均覆盖；Top10 有数量支撑；H1 有单独统计（结论为「很少」也算完成）。

### 6.1 需求优先级矩阵（周五评审核心）

| 需求 / 场景 | 用户规模（估） | 动机强度 | 使用频率 | 竞品满足度 | 2 个月可验证 | 综合优先级 |
|-------------|---------------|----------|----------|------------|--------------|------------|
| （由 PM 根据 T1–T5 填写） | 高/中/低 | 高/中/低 | 高/中/低 | 高/中/低 | 高/中/低 | P0/P1/P2 |

**打分说明（给非产品同学）**

| 维度 | 高 | 低 |
|------|----|----|
| 动机强度 | 不练有真实损失 | 可有可无 |
| 竞品满足度 | Speak 等已做得很好 | 用户普遍吐槽、竞品没做 |
| 2 个月可验证 | 脚本 + 对话 + 总结页可演示 | 依赖自研语音引擎 |

### 6.3 假设 H1（冲突 / 高压口语）周五表决

与其他方向 **同等标准**，不因 7.7 文档默认加分。

**窄市场三条件**（满足 **≥2 条** 可保留为候选方向之一）：

1. **访谈** ≥3 人有具体高压故事，且表示「愿意练」
2. **竞品矩阵** 中几乎没有合格的高压场景 + 反馈闭环
3. **T1 社媒 + T3 评论** 中「太温和 / 真实场景不够 / 不敢争」信号明显

| 表决结果 | 处理方式 |
|----------|----------|
| 3 条全过 | 保留为 **主推候选**；对外叙事用「高压脱敏 / Assertive English」，场景扩至谈判、维权、会议 |
| 过 1–2 条 | 保留为 **获客钩子或子模式**；产品内核绑定「教练复盘」等更通用能力 |
| 几乎不过 | **归档 backlog**；7.7 文档作灵感库，不删 |

## 十、站会与评审

### 每日站会（15 分钟）

只回答三句：

1. 昨天收集了什么 **证据**（链接 / 原话 / 条数）？
2. 今天计划？
3. 卡在哪（找不到人访谈 / 平台搜不到 / 竞品信息缺失）？

### 周五评审议程（2 小时）

| 时间 | 内容 |
|------|------|
| 0:00–0:20 | T1 用户需求证据摘要 |
| 0:20–0:35 | T2 竞品结论 + T3 评论 Top10 |
| 0:35–0:45 | T4 市场数据 + T5 趋势（可选） |
| 0:45–1:15 | 过需求优先级矩阵 |
| 1:15–1:45 | 假设 H1 表决 + 锁定 2–3 个方向 |
| 1:45–2:00 | 下周行动：是否进入 PRD / 场景脚本 / 原型（仍不做技术 Benchmark，除非方向明确要求） |

## 十一、质量红线（全员遵守）

1. **所有结论必须带证据**：用户原话、帖子链接、评论原文或数据来源链接  
2. **只写需求，不写方案**：本周不出现「我们要做 XX 功能」  
3. **不预设方向 D**：H1 单独统计，周五投票决定  
4. **不追求报告厚度**：可讨论的表格 > 漂亮 PPT  
5. **访谈避免全是熟人**：至少一半非亲密关系
