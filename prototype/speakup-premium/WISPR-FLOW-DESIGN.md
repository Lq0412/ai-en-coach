# Wispr Flow → SpeakUp 门户设计系统

> 提取日期：2026-07-22
>
> 观察页面：https://wisprflow.ai/
>
> 用途：复刻 Wispr Flow 的前端设计语言，并映射到 SpeakUp 产品验证门户。本文记录结构与规则，不复制对方品牌资产、Logo、插画或源代码。

## 1. Visual Theme & Atmosphere

Wispr Flow 的营销页不是常见的“蓝紫渐变 SaaS”。它使用编辑感强的超大衬线标题、暖米白底色、接近黑色的整段分区，以及淡紫、橙色作为少量高饱和强调色。大面积留白负责建立高级感，2px 实线边框和夸张圆角负责让页面保持友好、可触摸。

- 整体感觉：编辑感、直白、温暖、产品优先。
- 视觉密度：首屏极低，产品演示区中等，功能区较高。
- 品牌姿态：一个动作讲清价值，再用真实界面证明。
- 标志性元素：超大衬线标题、80px 大圆角深色分区、带边框按钮、胶囊选项、产品界面动画。

### SpeakUp 采用

- 采用“一个明确动作 + 一个真实产品演示”的首屏结构。
- 采用米白、近黑、淡紫、深绿、橙色的色彩角色。
- 采用大圆角整段分区和粗边框，不使用渐变发光。
- 用现有原型截图替换 Wispr 的产品动画；缺失画面明确标注占位。

### Theme Modes

Wispr Flow 首页没有全局明暗模式切换，而是在同一页面中交替使用亮色和深色分区。SpeakUp 沿用这种“区块主题”而不是增加主题切换。

### Shadows & Depth

- 主要深度来自边框、背景切换和局部错位，不靠大面积模糊阴影。
- 按钮：2px 深色边框，默认无阴影。
- 产品设备框可以使用一个实色错位阴影，避免玻璃感。
- focus ring：3px 蓝色外描边，和品牌装饰色分离。

## 3. Typography Rules

### Font Family

- Wispr 标题：`EB Garamond`，常规字重。
- Wispr 正文：`Figtree`。
- SpeakUp 中文标题映射：`Songti SC`, `STSong`, `Georgia`, serif。
- SpeakUp 中文正文映射：`PingFang SC`, `Microsoft YaHei`, system-ui。
- 英文回答与问题：`Georgia`, `Times New Roman`, serif，可强化编辑与口语稿件感。

### Hierarchy

| 角色 | 桌面 | 移动 | 字重 | 行高 | 字距 |
| --- | --- | --- | --- | --- | --- |
| Hero | 120px（观察值）/ SpeakUp 最高 116px | 56px（观察值）/ SpeakUp 47–68px | 400 | 0.85–0.96 | `-5%` |
| Section H2 | 64–84px | 48px | 400 | 0.95–1.02 | `-4%` |
| Card H3 | 32–40px | 30–34px | 400 | 1.05–1.15 | `-3%` |
| Lead body | 18–22px | 16px | 400 | 1.6–1.7 | normal |
| Label | 12–13px | 12px | 700 | 1.3 | `0.08–0.12em` |

### Principles

- 标题短，尽量使用动词和一个强对比。
- 中文标题用宋体产生编辑感，正文保持无衬线保证阅读。
- 不用粗黑大标题模拟力量感；力量来自字号、留白和对比。
- 一屏只允许一个最高层级标题。

## 4. Component Stylings

### Buttons and Links

观察值：主要按钮 `#F0D7FF` 背景、2px `#1A1A1A` 边框、12px 圆角、`16px 24px` 内边距、600 字重；hover 为 `scale(.98)`。

- 主按钮：淡紫填充、2px 黑边、12px 圆角。
- 次按钮：透明或米白填充，保留同样边框。
- 深色区次按钮：透明底、米白边和文字。
- 文本链接：下划线偏移 3–5px，不用灰色小箭头代替可点击性。

### Cards and Containers

- 普通信息卡：2px 边框，30–44px 圆角，无阴影。
- 整段分区：40–80px 圆角；桌面 80px，移动 42px 左右。
- 内边距：桌面 34–68px，移动 20–32px。
- 禁止卡片套卡片；产品截图设备框属于内容展示，不再包一层装饰卡。

### Inputs and Interactive Controls

- 输入框保持纸张表面和 2px 边框。
- 场景选择使用 999px 胶囊；观察到选中项使用淡紫填充并旋转约 `-4deg`。
- 所有触摸目标最少 42px 高。

### Navigation

- 桌面为品牌 / 中部链接 / 右侧 CTA 的三列结构。
- 移动隐藏中部链接，只保留品牌和 CTA。
- 当前门户不使用复杂下拉导航，避免在验证阶段分散注意力。

### Image Treatment

- 原型截图放入大面积彩色区，使用粗边框设备框。
- 截图不加多层渐变、光晕和玻璃模糊。
- 图片统一顶部对齐裁切；不把多张手机图硬挤在一屏。

## 5. Layout Principles

### Spacing System

- 基础单位：8px。
- 常用：12、20、28、34、44、64、86、128px。
- 观察到 Wispr section padding 主要为 96、128、160px 等大尺度。

### Grid & Container

- 最大内容宽度：1240px。
- Hero 文案最大宽度约 1030px，副文案 700px。
- 两栏演示比例不必 1:1，优先让产品画面和结果占更大空间。
- 功能区桌面三列，平板与手机单列。

### Border Radius Scale

- Micro：8–12px，用于按钮和标签。
- Standard：16–24px，用于输入和小模块。
- Large：34–48px，用于产品框和功能卡。
- Section：72–80px；移动缩至 38–42px。
- Pill：999px。

## 6. Depth & Elevation

| 层级 | 处理 | 用途 |
| --- | --- | --- |
| Flat | 纯背景，无边框 | 页面基础和文字区 |
| Ring | 2px 实线边框 | 按钮、胶囊、产品模块 |
| Product | 实线边框 + 单层错位阴影 | 手机原型截图 |
| Focus | 3px `#2D62FF` 外圈 | 键盘操作 |

### Depth Principles

- 深度首先依靠颜色和边界，不依靠模糊阴影。
- 同一模块最多一个阴影层级。
- 不使用玻璃拟态、霓虹发光、紫蓝渐变。

## 7. Do's and Don'ts

### Do

- 首屏只讲“模拟面试”一个动作。
- 把真实原型直接放进页面，缺失处标注占位。
- 使用大字号、强留白和单个强调色建立节奏。
- 用“原回答 / 反馈后”展示产品价值。
- 保留模拟面试、报告、复练之间的实际入口。

### Don't

- 不复制 Wispr 的 Logo、插画、动画文件或营销文案。
- 不在首屏并列面试、雅思、生活、职场四大入口。
- 不使用虚构用户评价、虚构数据和虚构合作 Logo。
- 不把完整 70 分钟计划作为第一次价值体验。
- 不把通用 Agent 当成差异化主卖点。

## 8. Responsive Behavior

| 名称 | 宽度 | 关键变化 |
| --- | --- | --- |
| Mobile | ≤640px | 单列、标题 47–68px、整段圆角 42px、按钮全宽 |
| Tablet | 641–980px | Hero 和产品演示改单列，功能卡改单列 |
| Desktop | ≥981px | 1240px 容器、两栏产品演示、三列功能卡 |

- 移动端保留大标题，但不把字号压成普通落地页。
- 浮动装饰胶囊在手机隐藏，避免遮挡内容。
- 面试官列表在手机隐藏时长列，保留角色和关注点。
- 导航移动端只保留品牌和主 CTA。

## 9. Agent Prompt Guide

### Quick Color Reference

- Primary CTA：`#F0D7FF`
- Background：`#FFFFEB`
- Heading：`#1A1A1A`
- Dark section：`#1A1A1A`
- Success / deep accent：`#034F46`
- Warm accent：`#FFA946`
- Border：`#1A1A1A` 或 18% alpha

### Quick Summary

创建一个受 Wispr Flow 启发、但属于 SpeakUp 的中文产品门户。使用暖米白背景、近黑深色分区、淡紫 CTA、宋体超大标题、2px 实线边框和 40–80px 大圆角。首屏只讲英文模拟面试，并马上展示现有产品原型。下面按模拟面试、即时反馈、报告与复练展开；缺少的社会证明使用明确占位，不制造完成度。

### Example Component Prompts

- Hero：用超大中文宋体标题“下一场英文面试，先和 SpeakUp 练一遍”，配一个真实语音回答和即时纠错演示，米白背景、淡紫产品舞台、2px 黑边。
- 模拟面试区：黑色 80px 圆角整段背景，左侧米白面试官计划，右侧短文案；面试阶段用胶囊切换，当前项淡紫并轻微旋转。
- 原型卡：三张高卡片，分别用橙、深绿、淡紫作为背景，下半部嵌入真实手机截图，状态明确标为“现有原型”或“待继续美化”。
- 占位区：使用虚线边框和空白引号，明确写“真实用户反馈待验证后补充”，不伪造 testimonial。

### Ready-to-Use Prompt

> 基于本设计系统修改 SpeakUp 门户。保留 Wispr Flow 的编辑感、大圆角分区、暖米白/近黑/淡紫配色和产品演示节奏，但不得复制其品牌资产。使用仓库现有模拟面试截图，首屏聚焦“下一场英文面试，先和 SpeakUp 练一遍”。缺失内容用可替换占位，保证 390px、768px、1440px 三档布局可用。

## Appendix A. Interaction Patterns

- 按钮 hover：200ms，`scale(.98)`。
- 胶囊选中：淡紫填充，轻微旋转。
- 页面滚动：普通纵向长页；整段分区通过背景和圆角切换节奏。
- 动效基调：短、直接、服务产品演示；尊重 `prefers-reduced-motion`。

## Appendix B. Content & Messaging

- 标题模式：一个直接动作，通常 8–18 个中文字符。
- 副文案：一句说明输入，一句说明结果。
- CTA：使用“开始模拟面试”“打开产品原型”等明确动词。
- 信任信息：只展示真实原型、真实访谈和真实数据。

## Appendix C. Evidence Notes

### 直接观察

- 桌面 1440px：H1 为 120px、`EB Garamond`、400 字重、102px 行高、`-6px` 字距。
- 手机 390px：H1 为 56px、47.6px 行高、`-2.8px` 字距。
- 主按钮：淡紫背景、2px 黑边、12px 圆角、`16px 24px` padding；移动端约 8px 圆角和更紧凑 padding。
- 页面主背景 `#FFFFEB`，主文字 `#1A1A1A`，深绿 `#034F46`，橙色 `#FFA946`。
- 大分区 CSS 变量包含 40、48、64、80px 圆角层级。
- 用例选项为 2px 米白边的胶囊；当前项淡紫填充并旋转约 `-4deg`。
- 按钮 hover 使用 `transform: scale(.98)`。

### 推断与 SpeakUp 取舍

- 推断：页面识别度主要来自字体反差、留白和结构色块，不来自装饰资产。
- 取舍：SpeakUp 先复刻结构和设计语法，不复刻复杂导航、动画数量和多行业用例规模。
- 取舍：社会证明区先占位，等待产品验证数据。
- 取舍：模拟面试是门户主入口；长期职业英语能力保留为后续扩展，不在第一版同时争夺注意力。
