## MS1 架构主干 Issue 骨架

```markdown
## 背景
产品 Proposal 已定稿（见 #xxx），需搭出系统架构主干空骨架。

## 要做的事
- 确定模块划分（列出所有模块及其职责）
- 定义模块间接口（接口为真实定义，实现可为桩）
- 定义核心数据模型
- 前后端 SDK 层定义（TypeScript 类型）
- 确保代码可编译、模块串联成立

## 验收标准
1. 所有模块目录已创建，package/模块文件已就位
2. 模块间接口定义完整（方法签名、参数类型）
3. 核心数据模型已定义（表结构/类型）
4. 项目可编译通过（无报错）
5. 能画出模块关系图并讲清协作逻辑
```
